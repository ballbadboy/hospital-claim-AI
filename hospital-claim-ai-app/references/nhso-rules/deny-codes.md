## Notes

### How C-Error Processing Works
1. **ส่งข้อมูล (Submit)**: หน่วยบริการส่งข้อมูลผ่านโปรแกรม e-Claim
2. **Validate (ตรวจสอบ)**: ระบบตรวจสอบข้อมูลอัตโนมัติ
3. **C-Error (ติด C)**: หากพบข้อผิดพลาด ระบบจะแจ้งรหัส C พร้อมคำอธิบาย
4. **แก้ไข (Fix)**: หน่วยบริการแก้ไขข้อมูลตามคำแนะนำ
5. **ส่งใหม่ (Resubmit)**: ส่งข้อมูลที่แก้ไขแล้วเข้าระบบอีกครั้ง
6. **ผ่าน A (Approved)**: ข้อมูลผ่านการตรวจสอบ เข้าสู่กระบวนการจ่ายชดเชย

### Status Codes in e-Claim System
| Status | Meaning |
|--------|---------|
| **A** | Approved - ผ่านการตรวจสอบ รอจ่ายชดเชย |
| **B** | Budget - อยู่ในงบประมาณ |
| **C** | C-Error - ติดข้อผิดพลาด ต้องแก้ไข |
| **D** | Deny - ปฏิเสธการจ่าย |
| **P** | Pending - รอดำเนินการ |
| **S** | Statement - ออก Statement แล้ว |

### e-Appeal Process
- เมื่อข้อมูลถูก Deny หรือต้องการแก้ไขข้อมูลที่ผ่าน A แล้ว ใช้ระบบ **e-Appeal**
- ดาวน์โหลดข้อมูลผ่านเมนู Appeal ใน http://eclaim.nhso.go.th
- แก้ไขข้อมูลแล้วส่งกลับเข้าระบบ
- รหัส **AP1** และ **EA1** เป็นรหัสแจ้งเพื่อทราบว่าข้อมูลเดิมถูกแทนที่ด้วยข้อมูลใหม่จาก e-Appeal
