## สรุปประเด็นสำคัญ

### 1. ผู้ป่วยในทั่วไป (ข้อ 5.1)
- **Base Rate คงที่**: 8,350 บาท/AdjRW (ในเขต)
- **ห้ามแบ่ง Episode**: ไม่สามารถแบ่งระยะเวลาการนอน รพ. เป็นช่วงๆ ในการรักษาต่อเนื่องกัน ระยะเวลา **270 วัน**
- หลัง 270 วัน → ส่งข้อมูลเสมือนจำหน่าย แล้วรับไว้ใหม่ได้
- **งบประมาณแบบมีเพดาน 5 หมวด**:
  1. บริการผู้ป่วยนอกทั่วไป
  2. บริการผู้ป่วยในทั่วไป
  3. บริการสร้างเสริมสุขภาพและป้องกันโรคพื้นฐาน
  4. บริการที่จ่ายตามรายการบริการ
  5. บริการที่บริหารแบบโครงการที่กำหนดวัตถุประสงค์เฉพาะ

### 2. อุปกรณ์และอวัยวะเทียม — Coronary (ข้อ 5.2)

#### 2.1 Pre Authorized บังคับ (ข้อ 5.2.1(1))
- **CCS (Chronic Coronary Syndromes)** ต้องขออนุมัติก่อนให้บริการ (**Pre Authorized**) **ทุกราย**
- ACS (Acute Coronary Syndromes) ยังไม่ต้อง Pre Auth

#### 2.3 เอกสารหลักฐานอุปกรณ์ (ข้อ 5.2.2)
- **บังคับส่งหลักฐาน** การใช้อุปกรณ์:
  - Sticker หรือ QR Code หรือ Bar Code หรือ Serial Number
- ส่งผ่าน **Standard API**
- **เริ่มบังคับ: 1 เมษายน 2569**

### 3. การจ่ายตามรายการบริการ (ข้อ 5.3)
- เปลี่ยนจาก: จ่ายตามอัตราที่กำหนด
- เป็น: **จ่ายตามจริงไม่เกินอัตราที่กำหนด**

### 4. OP อุบัติเหตุ/ฉุกเฉิน (ข้อ 5.4)
- เปลี่ยนเป็นจ่ายด้วย **ระบบคะแนน (Point System)** ตามราคารายการที่กำหนด

## ผลกระทบต่อโรงพยาบาล

### สำหรับ Cath Lab โดยเฉพาะ:
1. **CCS ต้อง Pre Auth ทุกราย** → ถ้าไม่ขออนุมัติก่อน = Deny แน่นอน
2. **เบิกอุปกรณ์เกินจำนวน = Deny** → เช่น DES สแตนเลส ≤1 ชุดเท่านั้น
3. **ต้องส่ง Serial/Barcode/QR ผ่าน API** ตั้งแต่ 1 เม.ย. 69 → ถ้าไม่ส่ง = HC09 Deny
4. **ห้ามแบ่ง Episode** ใน 270 วัน → staged PCI ต้องวางแผนดี

### Deny Risk เพิ่มขึ้น:
- HC09 (INST) → จำนวนอุปกรณ์เกิน / ไม่มี Serial-Barcode
- IP01 → CCS ไม่มี Pre Auth
- ค่าใช้จ่ายอุปกรณ์จ่ายตามจริงไม่เกินอัตรา → ราคาเกิน = ตัดเหลือเพดาน

## Quick Reference — เพดานอุปกรณ์ที่ใช้บ่อย

| สถานการณ์ | อุปกรณ์ที่เบิกได้ |
|-----------|-------------------|
| **Diagnostic CAG only** | Diag Cath ≤3, Diag Wire ≤2, Angio Cath ≤3, Sheath ≤3 |
| **PCI 1 vessel (DES อัลลอยด์)** | Guiding ≤2, PTCA Wire ≤3, Balloon ≤5, DES-A ≤3, Sheath ≤3, Closure ≤4 |
| **PCI multi-vessel** | เหมือนข้างบน แต่รวมทุก vessel ต่อครั้ง |
| **PCI + CTO** | เพิ่ม CTO Wire ≤3, CTO Balloon ≤2 |
| **PCI + Rotablation** | เพิ่ม Rota Burr ≤2, Advancer ≤1, Rota Wire ≤1 |

## การนำไปใช้ในระบบ Claim

### Validation Rule เพิ่มเติม:
```
IF diagnosis = CCS (I25.x) THEN
  CHECK Pre_Auth_Code EXISTS → ถ้าไม่มี = DENY

IF device_count > max_per_procedure THEN
  FLAG "เกินจำนวนที่กำหนด" → DENY

IF service_date >= 2569-04-01 THEN
  CHECK device_serial OR barcode OR qr_code EXISTS → ถ้าไม่มี = HC09 DENY
```

### อ้างอิงในหนังสืออุทธรณ์:
> ตามประกาศสำนักงานหลักประกันสุขภาพแห่งชาติ เรื่อง หลักเกณฑ์การดำเนินงาน
> และการบริหารจัดการกองทุนหลักประกันสุขภาพแห่งชาติ และค่าใช้จ่ายเพื่อบริการ
