# คู่มือระบบ e-Claim สปสช.

> Source: การขอรับค่าใช้จ่ายผ่านระบบโปรแกรม E-CLAIM — สปสช. เขต 7 ขอนแก่น
> URL: https://eclaim.nhso.go.th/Client

## 1. ภาพรวมการทำงานของระบบ

1. ระบบ e-Claim ใหม่เป็นระบบแบบ **web online**
2. การเข้าใช้งานระบบ เข้าใช้งานโดยใช้ **USER ที่ สปสช. ออกให้** (USER ของเว็บ e-Claim)
3. การบันทึกข้อมูล มี 2 รูปแบบ คือ
   1) การบันทึกข้อมูลแบบ **คีย์รายเคส**
   2) การ **นำเข้าข้อมูล (16 แฟ้ม)**
4. การส่งข้อมูล
5. การรับข้อมูลผลการตรวจสอบ REP
6. การแก้ไขข้อมูลกรณีไม่ผ่านการตรวจสอบ
7. การออก Statement ของข้อมูลที่ผ่านการตรวจสอบ
8. รอบการโอนเงิน

---

## 2. กระบวนการประมวลผลจ่ายชดเชยในระบบ e-Claim

### Flow:
```
รพ.สต./รพ.แม่ข่าย (0) → HIS DATABASE
       ↓
(1) e-Claim DATABASE (รับและตรวจสอบเบื้องต้น)
       ↓ จันทร์/พุธ/ศุกร์
(2) e-Claim DATABASE สปสช. (ประมวลผล/ตรวจสอบ)
       ↓
(3) STM วางฎีกา
       ↓ (4) PPFS → Mark Audit → จ่ายเพิ่ม → เรียกคืน → PPFS DATABASE
(5) ↓
(6) E-FORM → SMT DATABASE (โอนเงิน)
       ↓
(7) NHSO Budget
```

### รอบการประมวลผล:
- **รับข้อมูล:** จันทร์ / พุธ / ศุกร์
- **PPFS audit:** รอบแรกกลางเดือน / รอบหลังสิ้นเดือน
- **โอนเงิน:** วันที่ 22 / 7 เดือนถัดไป

- **ค้นหาข้อมูล AUTHEN** — ตรวจสอบ Authen Code

## 5. การจัดการข้อมูล (ก่อนเริ่มบันทึก)

### เมนูจัดการข้อมูล:
1. **แพทย์** — จัดการรายชื่อแพทย์ (รหัสแพทย์, คำนำหน้า, ชื่อ, นามสกุล, สาขา)
2. **โรงพยาบาล**
3. **วินิจฉัยโรค** — ICD-10 codes
4. **การทำหัตถการ** — ICD-9-CM codes
5. **อวัยวะเทียม/อุปกรณ์บำบัดรักษาโรค**
6. **บริการอื่นๆ ก่อไม่จัดหมวดหมู่**
7. **การตรวจสอบความถูกต้องและวิธีแก้ไข**
8. **ข้อความแจ้งเตือนข้อผิดพลาด**
9. **ยาของหน่วยบริการ (Drug Catalog)** — รหัสยา Hosp Drug Code, ประเภทยาและเวชภัณฑ์, รหัสยา TMT, ชื่อยาสามัญ, ชื่อทางการค้า, DSF Code, ลักษณะยา, ปริมาณต่อหน่วยยา
10. **การตรวจทางห้องปฏิบัติการของหน่วยบริการ (LAB Catalog)**

### Drug Catalog — ฟิลด์สำคัญ:

| ฟิลด์ | ตัวอย่าง |
|-------|---------|
| รหัสยา Hosp Drug Code | ABA100E |
| ประเภทยาและเวชภัณฑ์ | 1 |
| รหัสยา TMT | 1041959 |
| ชื่อยาสามัญ | abacavir |
| ชื่อทางการค้า | ABACAVIR |
| DSF Code | 1 |
| ลักษณะยา | film-coated tablet |
| ปริมาณต่อหน่วยยา | 300 mg |

**ปุ่ม "เชื่อมโยงข้อมูล"** — ใช้เชื่อม Drug Catalog กับรหัส TMT กลาง

## 6. การบันทึกข้อมูล (Key Case)

### ขั้นตอน:
1. เลือก **ผู้ป่วยนอก → บันทึกผู้ป่วยนอก** (หรือ ผู้ป่วยใน)
2. ระบบแสดงรายการ → กด **+ สร้าง**

### 7 Tabs การบันทึก:

| Tab | ชื่อ | ข้อมูล |
|-----|------|-------|
| F1 | ข้อมูลทั่วไป | วันที่เข้ารับบริการ, เวลา, HN, ประเภทบัตร, หมายเลขบัตร, ตรวจสอบสิทธิ, เพศ, วันเกิด, อาชีพ, สิทธิประโยชน์, Authen Code |
| F2 | การวินิจฉัยโรค | วินิจฉัยโรคหลัก (ICD-10), รหัสแพทย์ผู้รักษา, รหัสแผนกที่รับบริการ, การวินิจฉัยโรครอง (ICD-10 เพิ่มได้หลายรายการ) |
| F3 | การผ่าตัดหัตถการ | รหัสผ่าตัด/หัตถการ (ICD-9-CM), รหัสแพทย์ผู้ทำการผ่าตัด, วันที่, เวลา, seq, รหัสแผนกที่รับบริการ, ราคาค่าบริการ |
| F4 | ค่าใช้จ่ายสูง | อวัยวะเทียม/อุปกรณ์ |
| F5 | มะเร็ง | ข้อมูลเฉพาะผู้ป่วยมะเร็ง |
| F6 | บริการเฉพาะ | บริการพิเศษ |
| F7 | ค่ารักษาพยาบาล | ค่ายา, ค่าเวชภัณฑ์ |

## 7. การบันทึกสิทธิประโยชน์

1. ประชาชนไทยทุกสิทธิ สามารถส่งเบิกโดยเลือกสิทธิประโยชน์เป็น **UCS** ได้ หรือ
2. ส่งเบิกตามสิทธิจริงของผู้ป่วย ประกอบด้วย
   - **OFC** : สิทธิข้าราชการสังกัดกรมบัญชีกลาง (ข้าราชการสังกัดอื่นส่งเบิกในสิทธิ UCS)
   - **LGO** : สิทธิ อปท.
   - **SSS** : สิทธิประกันสังคม
   - **BKK** : สิทธิข้าราชการ กรุงเทพมหานคร
3. สิทธิอื่นๆ นอกเหนือจากที่ระบุในข้อ 2 ต้องส่งเบิกในสิทธิประโยชน์ **UCS**

## 8. การตรวจสอบสิทธิ

### ขั้นตอน:
1. กรอก **วันที่เข้ารับบริการ** + เวลา
2. กรอก **เลขบัตรประจำตัวผู้ป่วย (HN)**, ประเภทบัตร, หมายเลขบัตร
3. กด **"ตรวจสอบสิทธิ"**
4. ระบบแสดง: หมายเลขบัตร, ชื่อผู้ป่วย, เพศ, วันเกิด
5. แสดง **รายการสิทธิ**: ลำดับที่, สิทธิที่ใช้หลัก, ประเภทสิทธิย่อย, ความสัมพันธ์, วันที่ออกบัตร/เริ่มใช้สิทธิ, หน่วยบริการ
6. กด **"เลือกสิทธินี้"**
7. ระบบแสดงข้อมูลต่างๆ ของผู้ป่วยให้อัตโนมัติ รวมถึง **รหัส Authen Code / เลขอนุมัติ** (เช่น PP1007991295)

## 12. ปฏิทินการออก Statement

### สำหรับข้อมูลผู้ป่วยนอก ในระบบหลักประกันสุขภาพ ปีงบประมาณ 2566

รอบ Statement ออกทุกครึ่งเดือน:
- เดือนที่ให้บริการ → วันที่ตัดยอด STM → วันที่ส่งข้อมูลครบ → รอบที่ออก Statement

ตัวอย่าง:
- บริการ ต.ค. 65 → STM 15/10/65, 31/10/65 → Statement ออก 18/10/65, 3/11/65, 18/11/65...
- วันที่หน่วยบริการได้รับเงินโอน: 21/10/65, 7/11/65, 22/11/65...

## สรุปสำหรับ AI Claim Validator

### สิ่งที่ต้องตรวจสอบก่อนส่ง e-Claim:

1. **สิทธิ์ถูกต้อง** — UCS, OFC, LGO, SSS, BKK
2. **Authen Code** — ต้องมี (เช่น PP1007991295)
3. **PDx (F2)** — ICD-10 ถูกต้อง, specific พอ
4. **SDx (F2)** — ครบถ้วน, CC/MCC ที่ควรมี
5. **Procedure (F3)** — ICD-9-CM ตรงกับ Dx
6. **Drug Catalog** — TMT code ตรง, มีใน Drug Catalog ของ รพ.
7. **16 แฟ้ม** — ครบถ้วน, format ถูกต้อง
8. **C Error** — ตรวจก่อนส่ง (download เอกสาร C error จากระบบ)
9. **Timing** — ส่งทันรอบ STM เพื่อออก Statement
