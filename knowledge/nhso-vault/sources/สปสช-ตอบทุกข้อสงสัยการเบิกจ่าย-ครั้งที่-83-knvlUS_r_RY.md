# สปสช. ตอบทุกข้อสงสัยการเบิกจ่าย ครั้งที่ 83

**Channel:** สปสช. | **Duration:** 1:10:08 | **Ingested:** 2026-04-13

> บันทึกการประชุม Live ที่เริ่มหลัง 13:00 น. เล็กน้อย (ย้ายเข้าสตูดิโอใหม่) วิทยากรหลัก: ท่านรองเลขาธิการ, อาจารย์อัครัต (OSS/Reactivate), อาจารย์มนพ, อาจารย์กฤต (Zero C)

## Summary

ครั้งที่ 83 จัดขึ้นก่อนวันที่ 12 มกราคม 2569 เพียงไม่กี่วัน ซึ่งเป็นวันเปิดตัวระบบหน่วยนวัตกรรม **Version 2** อย่างเป็นทางการ เนื้อหาหลักแบ่งเป็น 3 หัวข้อ: (1) อัปเดตระบบใหม่ของหน่วยบริการนวัตกรรม 7 ประเภท รวมถึงระบบโควต้ารายสัปดาห์, Face Recognition, และ Line Official; (2) ขั้นตอน Reactivate ใน OSS และปัญหาที่พบบ่อย (สภาการพยาบาลยัง API ไม่สมบูรณ์, GPSต้องทำที่คลินิก, CPP version ใหม่); (3) นโยบายใหม่ **Zero C** ที่เปลี่ยนวิธีจัดการข้อมูลที่ติด C-code จากการระงับทั้งเคสมาเป็นจ่าย 0 บาทรายไอเทม เพื่อแก้ปัญหาสภาพคล่องหน่วยบริการ

ช่วงท้ายตอบคำถาม Google Form 3 ข้อ: ผลกระทบจากการยกเลิก re-run (เลื่อนตอบสัปดาห์หน้า), ร้านยาโมเดลที่ติด C30/ปิดสิทธิ์ (เงินส่วน PP โอนแล้ว, โมเดล 3 อยู่ระหว่างโอน), และโรงพยาบาลภูผาม่านที่ติด C936/C179 (ต้องย้ายสิทธิ์ใน SRM ก่อนส่งเบิก)

## Key Points

- **12 มกราคม 2569** คือวันเปิดระบบนวัตกรรม Version 2 (ทดสอบ Line NHSO วันที่ 7 ม.ค. 2569)
- ยกเลิกจำกัด 2 ครั้ง/คน/ปี — ให้บริการตามความจำเป็นทางการแพทย์ภายใต้โควต้ารายสัปดาห์
- โควต้าเริ่มรีเซ็ตทุกวันจันทร์ เวลา **18:00 น.** ของสัปดาห์ก่อน
- ประชาชนไม่ต้องจองก็ได้ — เดิน Walk-in แล้วสแกนใบหน้าได้เลย (Line OA เป็นทางเลือก)
- เด็ก < 15 ปี และผู้สูงอายุ > 60 ปี สแกนหน้าอาจไม่ผ่าน ให้ใช้ Line แทน
- **Reactivate**: ต้องทำที่คลินิกเท่านั้น (พิกัด GPS ต้องแม่นยำ, รัศมีครึ่งกิโล)
- สภาการพยาบาลยัง API ไม่สมบูรณ์ ณ วันที่ประชุม — คลินิกพยาบาล reactivate ยังไม่ได้
- **CPP version ใหม่**: เวอร์ชันเก่าหมดอายุ 31 ธ.ค. 68 ต้องบันทึกบุคลากรใหม่ทั้งหมด
- **Zero C**: ระบบใหม่จ่าย 0 บาทรายไอเทม (แทนติด C ทั้งเคส) พร้อมแจ้งเหตุผลแบบ near real-time
- C30 backlog: สปสช. cleansing ข้อมูล คาดเปิดแก้ไขภายในมกราคม-กุมภาพันธ์ 2569
- C936: เกิดจากไม่ย้ายสิทธิ์ใน SRM ก่อนส่งเบิก → C179 เมื่อส่งซ้ำ
