## Summary

Live ครั้งที่ 94 ของ สปสช. ตอบทุกข้อสงสัยการเบิกจ่าย ออกอากาศช่วงบ่ายวันศุกร์ เน้น 4 หัวข้อหลักที่เกิดขึ้นในสัปดาห์ที่ผ่านมา ได้แก่ ไทม์ไลน์การปรับปรุงข้อมูลบุคลากรสำหรับหน่วยบริการที่ reactivate, ปัญหา Error N019 ในระบบหน่วยนวัตกรรม, การเปลี่ยนระบบ OSS Two-Factor Authentication (2FA) แบบกะทันหัน และแนวทางการยืนยันตัวตนของกลุ่มเปราะบาง (เด็ก/ผู้สูงอายุ) รวมถึงการหยุดใช้งานระบบ CKD DMIS ชั่วคราวเพื่ออัพเกรดความปลอดภัย

### Q1: หน่วยบริการที่ reactivate แล้ว — ไทม์ไลน์แก้ไขข้อมูลบุคลากรทำอย่างไร?

**บริบท:** หน่วยนวัตกรรม 7,800+ แห่ง ได้ reactivate เข้ามาแล้ว (เริ่มให้บริการตั้งแต่ 12 ม.ค.) แต่หลายแห่งต้องการแก้ไข/เพิ่ม/ลบชื่อบุคลากร

**คำตอบ — ไทม์ไลน์รายสัปดาห์:**
1. **วันจันทร์ ก่อนเที่ยง** — หน่วยบริการแจ้งการแก้ไขข้อมูลบุคลากรไปที่ **สปสช. เขต** ที่ตนสังกัด (แจ้งได้ตลอดเวลา เขตรวบรวมส่งส่วนกลางภายในเที่ยงวันจันทร์)
2. **วันอังคาร ก่อนเที่ยง** — สำนักทะเบียน สปสช. ตรวจสอบความถูกต้อง ส่งต่อให้บริษัทผู้รับผิดชอบ
3. **วันพุธ ไม่เกิน 10:00 น.** — บริษัทดำเนินการแก้ไขในระบบเรียบร้อย

> **สรุป:** แจ้งเขตได้ตลอดเวลา → เขตรวบรวมส่งส่วนกลางทุกจันทร์เที่ยง → ส่งบริษัทอังคารเที่ยง → แก้เสร็จพุธ 10:00 น. (วนซ้ำทุกสัปดาห์)

**หมายเหตุ:** ข้อมูลบุคลากรต้องเชื่อมกับสภาวิชาชีพ + แจ้ง Longitude/Latitude ของหน่วย (ใช้กับ FVS Face Verification)

### Q2: Error N019 (IN019) คืออะไร และแก้ไขอย่างไร?

**บริบท:** หน่วยบริการหลายแห่งพบ Error N019/IN019 ในระบบหน่วยนวัตกรรม ไม่ได้รับเงินชดเชย

**คำอธิบาย:**
- **IN019 = Innovation 019** คือ Payment Type Code สำหรับกลุ่มบริการ **OP (Outpatient) ของหน่วยนวัตกรรม** (7 วิชาชีพ) ที่มีการคำนวณโควต้า
- Payment Type หลัก: `PG019` = OP นวัตกรรมที่มีโควต้า | `PG006` = PP (ส่งเสริมป้องกัน) ที่ไม่มีโควต้า

**สาเหตุของ Error:**
- หน่วยบริการส่งปิดสิทธิ์ **หลายรอบในครั้งเดียวกัน** (เช่น ทำ 2 บริการพร้อมกัน) ระบบดึงเฉพาะ **รายการสุดท้าย** มาประมวลผล
- ถ้ารายการสุดท้ายเป็น `PG019` แต่บริการแรกเป็น PP (`PG006`) → บริการ PP อาจถูกปฏิเสธด้วย IN19

**การแก้ไข:**
- สปสช. จะ **re-run ข้อมูล** ในวันจันทร์ถัดไป เพื่อดึงรายการที่ติด IN19 มาจ่ายใหม่
- กรณีที่ทำ Face Scan (FVS) ไว้ **2 ครั้งแล้ว** — สปสช. จะดึงมาจ่ายเองโดยอัตโนมัติ **ไม่ต้องอุทธรณ์**

**แนวทางป้องกัน:**
- เลือก Payment Type ให้ถูกต้อง: บริการ C (ล้างแผล/กายภาพ) → `PG019`; บริการ PP → `PG006`
- สแกนใบหน้า (FVS) ก่อนทุกครั้ง อย่างน้อย 2 ครั้ง เพื่อให้ระบบเก็บ log

### Q3: OSS Two-Factor Authentication (OSS-2FA) เปลี่ยนกะทันหัน — ทำอย่างไร?

**บริบท:** ช่วงวันศุกร์-อาทิตย์ก่อน Live ระบบเดิม (Data Center / Single Factor) ใช้ไม่ได้ หน่วยบริการต้องใช้ระบบใหม่ **OSS (One Stop Service) + 2FA** โดยไม่ได้แจ้งล่วงหน้าเพียงพอ

**พื้นหลัง — ระบบเก่า vs. ระบบใหม่:**

| รายการ | ระบบเก่า (Data Center) | ระบบใหม่ (OSS) |
|--------|------------------------|-----------------|
| Username | ตัวเลข 12-13 หลัก | ตั้งเองได้ (ตัวอักษร+ตัวเลข) |
| Authentication | Single Factor (User+Password เท่านั้น) | Two Factor (User+Password + ThaiD สแกน QR) |
| ความปลอดภัย | ต่ำ | สูง (เหมือน Mobile Banking ปัจจุบัน) |

**ขั้นตอนการใช้งาน OSS-2FA:**
1. **ดาวน์โหลดแอป ThaiD** จาก Google Play / App Store (ของกรมการปกครอง)
2. **ลงทะเบียน ThaiD** — ถ่ายรูปบัตรประชาชน + สแกนใบหน้า + ตั้ง Passcode 7 หลัก
3. **เข้าระบบ OSS** — กด Login ด้วย ThaiD → สแกน QR code ที่แอป ThaiD บนมือถือ → เข้าสู่ระบบ
4. **ตรวจสอบบทบาท (Role)** — เฉพาะผู้มีอำนาจลงนาม หรือผู้ที่ได้รับมอบอำนาจ เท่านั้นที่สร้าง User / จัดการบทบาทในระบบ OSS ได้
5. **เพิ่ม User ใหม่บน OSS** — ถ้ายังใช้ User ของ Data Center อยู่ ให้สร้าง User ใหม่บน One Stop Service

**ข้อควรระวัง:**
- **1 ThaiD = 1 มือถือ** — ห้ามให้คนอื่นสแกน QR แทน เพราะระบบเก็บ log ชื่อผู้สแกน
- **อย่าเปิด Session ค้าง** — ออกจากระบบทุกครั้งหลังใช้งาน เพราะเข้าพร้อมกัน 2 เครื่องจะมีปัญหา
- หากมี **2 หน่วยบริการ** ใน User เดียว → สลับโปรไฟล์ในหน้า OSS ก่อนเข้าโปรแกรมอื่น

**สาเหตุที่ FVS สแกนช้า/ติด-ดับ:** ระบบ ThaiD ของกรมการปกครองอาจ rate limit เพราะมีผู้ใช้งานพร้อมกันมาก อยู่ระหว่างเจรจาเพิ่ม limit

### Q4: กลุ่มเปราะบาง (เด็ก/ผู้สูงอายุ) — ใช้ระบบ New Authentication อย่างไรไม่ให้สะดุด?

**บริบท:** ระบบ FVS (Face Verification Service) สแกนใบหน้าใช้ยืนยันตัวตนผู้รับบริการ แต่กลุ่มเปราะบางบางรายสแกนไม่ผ่าน

**ลำดับขั้นตอนสำหรับกลุ่มเปราะบาง:**

| ลำดับ | วิธี | เงื่อนไข |
|-------|------|-----------|
| 1 | **สแกนใบหน้า (FVS)** | วิธีหลัก — ทุกคนต้องลองก่อน **อย่างน้อย 2 ครั้ง** |
| 2 | **จองผ่าน Line OA** | กรณีสแกนไม่ผ่าน 2 ครั้ง |
| 3 | **เสียบ Smart Card** | กลุ่มพิเศษ + กรณีสแกนไม่ผ่านจริง |

**กลุ่มพิเศษที่ใช้วิธีที่ 3 ได้:**
- เด็กอายุ **ต่ำกว่า 7 ปี** (ไม่มีบัตรประชาชน ไม่มีข้อมูลในฐาน)
- เด็กอายุ **7–14 ปี** ที่สแกนไม่ผ่าน (ลอง 2 ครั้งก่อน)
- ผู้สูงอายุ/ผู้พิการ ที่สแกนไม่ผ่าน (ลอง 2 ครั้งก่อน)

**ขั้นตอนการยืนยันตัวตนแทน (วิธีที่ 3):**
1. เข้าระบบ (endpoint ปิดสิทธิ์)
2. เลือก "ยืนยันตัวตนแทนผู้เข้ารับบริการ"
3. ใส่เลขบัตรประชาชนของผู้รับบริการ → ค้นหา
4. เลือก **เหตุผล** ที่ไม่สามารถสแกนได้ (เด็ก / ผู้สูงอายุ / ผู้มีปัญหาด้านใบหน้า)
5. เลือก **ความสัมพันธ์** ของผู้ยืนยันแทน (พ่อ/แม่/ปู่/ย่า/ผู้ดูแล)
6. เสียบ Smart Card ของผู้รับบริการก่อน → ถ้าไม่มี ใช้บัตรผู้ดูแล/ปกครอง
7. กรอกข้อมูลปิดสิทธิ์ตามปกติ → ติ๊กยืนยันว่าเป็นบริการนวัตกรรม → บันทึก

> **สำคัญ:** ต้องสแกน 2 ครั้งก่อนเสมอ เพื่อเก็บ log ว่าสแกนไม่ผ่านจริง หากไม่มี log 2 ครั้ง ระบบอาจปฏิเสธการจ่าย

### Q5: CKD DMIS หยุดใช้งานชั่วคราว — เหตุผลและวิธีปฏิบัติ

**บริบท:** ระบบ CKD DMIS (ระบบสารสนเทศเพื่อบริการผู้ป่วยไตวายเรื้อรังและผู้ป่วยระยะสุดท้าย) หยุดให้บริการชั่วคราว

**เหตุผล:** ระบบ CKD DMIS พบปัญหาด้านมาตรฐานความปลอดภัย จึงต้องยกระดับ (upgrade) ระบบ Authentication ให้เข้ากับ OSS-2FA เช่นเดียวกับระบบอื่น

**ช่วงเวลาปิด:** 27–29 มีนาคม (ประกาศบนเว็บไซต์สปสช.)

**เปิดอีกครั้ง:** วันจันทร์ เวลา 8:00 น.

**วิธีปฏิบัติระหว่างปิดระบบ:**
1. **ปิดสิทธิ์วันเสาร์-อาทิตย์** ตามปกติ แต่ไม่ต้องบันทึกในระบบ
2. **วันจันทร์หลังระบบเปิด** → บันทึกข้อมูลย้อนหลังในระบบ CKD DMIS
3. สปสช. เลื่อนการดึงข้อมูลเพื่อประมวลผลจ่ายเงิน **จากวันที่ 1 เป็นวันที่ 5** ของเดือน (เพื่อให้หน่วยมีเวลาบันทึกย้อนหลังให้ครบ)

**การเตรียมความพร้อมสำหรับ CKD DMIS ระบบใหม่:**
1. ดาวน์โหลดแอป ThaiD
2. ตรวจสอบว่า PID (เลขบัตรประชาชน) ที่ใช้ใน ThaiD ตรงกับที่ผูกใน User บน OSS
3. ตรวจสอบ/เพิ่มบทบาท (Role) สำหรับ "ระบบสารสนเทศเพื่อการให้บริการผู้ป่วยไตวายและระยะสุดท้าย" ใน OSS

## Q&A จาก Google Form (12 ข้อ)

**ข้อ 1 — ทำแผลเด็ก < 15 ปี, สแกนหน้าไม่ผ่าน → ไม่ได้รับเงิน**
→ ต้องสแกน 2 ครั้งก่อนเพื่อเก็บ log แล้วใช้วิธียืนยันตัวตนแทน (วิธีที่ 3) ตามขั้นตอน Q4

**ข้อ 2 — อินซูลิน (Insulin) บัตรทองใช้ได้มั้ย?**
→ อยู่ใน **Free Schedule 2** เบิกได้เป็นครั้งๆ ตามรหัส **A38614** ตรวจสอบใน Drug Catalog ของโรงพยาบาล (TMT Code) ผู้รับบริการบัตรทองไม่ต้องชำระเงิน

**ข้อ 3 — ขอ One-page สรุปแนวทางขอรับค่าบริการแต่ละกองทุน**
→ สปสช. มีเอกสารสรุปแนวทางขอรับค่าใช้จ่ายประจำปี ~400 หน้า แยกตามกองทุนย่อย ดาวน์โหลดได้จาก Google Drive (ลิงก์ส่งในแชทไลฟ์)

**ข้อ 4 — IP number ออกแล้ว แต่เงินยังไม่เข้า**
→ งวด 16 มีนาคม มีการโอนแล้ว แต่เนื่องจากระบบโควต้าทำให้ประมวลผลช้าขึ้น วางฎีกาล่าช้า (จากพุธเป็นพฤหัส-ศุกร์ บางแห่ง)

**ข้อ 5–6 — IN019 ติดแต่ทำถูกขั้นตอน**
→ ระบบจะรีรันวันจันทร์ ถ้ามี FVS log 2 ครั้ง จะจ่ายโดยอัตโนมัติ ไม่ต้องอุทธรณ์

**ข้อ 7 — คลินิกแพทย์แผนไทย โควต้าเป็นอย่างไร?**
→ บริการ C มีโควต้า แต่บริการอื่นๆ ให้บริการตามความจำเป็นของผู้รับบริการ/มาตรฐานวิชาชีพ

**ข้อ 8 — บริษัทมี 2 หน่วยบริการ 1 User เข้าได้ทั้ง 2 มั้ย?**
→ ให้ **สลับโปรไฟล์** ในหน้า OSS ก่อนเข้าโปรแกรมอื่น (ระบบจะจำ transaction สุดท้าย) หรือแยก PID 1 คน ต่อ 1 H-Code

**ข้อ 9 — Smart Money Transfer ยอดคงค้างผิดปกติ (เพิ่มขึ้นเรื่อยๆ)**
→ ให้ยึดจากไฟล์ PDF ในแต่ละบรรทัดของวันที่ปรากฏ ไฟล์ PDF จะไม่เปลี่ยน ดูรายละเอียดหักกลบจาก statement

**ข้อ 10 — เงิน PP ค้างชะลอถึงเมื่อไหร่?**
→ ชะลอจนกว่าหักหนี้ก้อนอื่นครบ (Smart Money Transfer ระบบสามารถหักข้ามกองทุนได้แต่ขึ้นกับหลักเกณฑ์ของสปสช. เขต)

**ข้อ 11 — เข้า OSS ได้แต่เข้า DMIS/Rate ไม่ได้**
→ ตรวจสอบโปรไฟล์ที่ใช้อยู่ว่าตรงกับระบบที่ต้องการเข้าหรือเปล่า สลับโปรไฟล์ให้ตรงก่อนเข้า

**ข้อ 12 — เด็กไม่ได้อยู่กับพ่อแม่ สแกนไม่ผ่าน ติด IN19 มา 3 งวด**
→ ต้องสแกน 2 ครั้งก่อนเสมอ แล้วใช้บัตรของผู้รับบริการก่อน ถ้าไม่มีค่อยใช้บัตรผู้ดูแล/ปกครอง
