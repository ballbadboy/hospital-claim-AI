# สปสช.ตอบทุกข้อสงสัยการเบิกจ่าย (ครั้งที่ 74)

**Channel:** สปสช. | **Duration:** 1:02:58 | **Ingested:** 2026-04-13

## Summary

วิดีโอนี้เป็นรายการสด สปสช. ตอบทุกคำถามเกี่ยวกับการเบิกจ่ายเงินชดเชยค่าบริการ ครั้งที่ 74 วันศุกร์ที่ 24 ตุลาคม ดำเนินรายการโดย อาจารย์มานพ โยเฮือง ร่วมกับท่านรอง เนื้อหาหลักเน้นการแก้ปัญหาที่เกิดจากเหตุการณ์ระบบทะเบียนสิทธิ์ล่ม (26–28 กันยายน) ที่ทำให้ข้อมูลสิทธิ์ผิดพลาดครั้งใหญ่ ส่งผลให้เคสจำนวนมากถูกปฏิเสธด้วยรหัส C38 (ต่อมาเรียก C438) ทั้งในระบบ MC Care, กรุงไทย Digital Health Platform และ New Attend

นอกจากนี้ยังมีการอภิปรายเรื่องกระบวนการตรวจสอบยอดโอนเงินผ่าน [[concepts/Seamless-DMIS]] และ [[concepts/Smart-Money-Transfer-โอนเงินอัจฉริยะ]] การแก้ไข C-Code ประเภทต่างๆ ก่อนสิ้นปีงบประมาณ 2568 และปัญหาเรื่องการปิดสิทธิ์ที่หน่วยบริการสะท้อนว่ายุ่งยากเกินไป สปสช. รับฟังและชี้แจงว่าเป็นกลไก check-and-balance เพื่อยืนยันว่าประชาชนได้รับบริการจริง

ไฮไลต์สำคัญที่พูดถึงในรายการ ได้แก่: รีรันข้อมูลผู้ป่วยใน 10 เดือนครึ่งด้วย Base Rate จริง (7,600–7,800 บาท), Dashboard CD9 สำหรับดูรายการติด C ระดับเขตและรายโรงพยาบาล, วิธีการตรวจสอบ IEP ผ่าน Seamless for DMIS, การอุทธรณ์ C350 กรณีเวลาปิดสิทธิ์ไม่สอดคล้อง และกระบวนการชะลอโอนเงิน (Delay Transfer) สำหรับหน่วยบริการที่มีพฤติกรรมผิดปกติ

## Key Points

- **รหัส C438 (C38)** — เกิดจากการย้ายระบบฐานทะเบียนสิทธิ์ช่วง 26–28 กันยายน ทำให้สิทธิ์ผู้รับบริการแปลงเป็นสิทธิ์ว่างหรือสิทธิ์อื่น ระบบทุกช่องทาง (Care, กรุงไทย DHP, New Attend) ปฏิเสธพร้อมกัน — สปสช. จะนำข้อมูลชุดนี้ประมวลผลใหม่และแจ้งงวดโอน
- **รีรัน IP Base Rate** — วันที่ 17 ต.ค. สปสช. ออกรายงานการจ่ายรอบ Re-run ข้อมูล IP ตั้งแต่ 16 ก.ย. 2567 ถึง 31 ก.ค. 2568 (10 เดือนครึ่ง) ด้วย Base Rate จริงรายเขต ได้ 7,600–7,800 บาท เฉลี่ยใกล้ 7,800 บาท บางหน่วยได้เลขติดลบ (ไม่ได้เรียกคืน — บัญชีไว้ก่อน)
- **เลขงวด (IEP Number)** — ใช้เป็น "กล่อง" อ้างอิงหลักในการตรวจสอบว่าข้อมูลบริการถูกประมวลผลหรือยัง ดูในระบบ Seamless for DMIS → ไอคอนบัตรประชาชนใบเดียว → รายงาน IEP Summary
- **วิธีตรวจสอบโอนเงิน** — เข้า nhho.go.th → บริการออนไลน์ → NHO Budget → รายงานโอนเงินกองทุน → ใส่รหัสหน่วยบริการ → ค้นเลขงวด → เทียบยอดเงิน
- **Dashboard CD9** — ระบบ Dashboard สารสนเทศข้อมูลปฏิเสธการจ่าย ดูได้รายเขตและรายโรงพยาบาล แสดงสถิติ C-Code และ 9-Code ครบทุกตัว พร้อมคำอธิบายวิธีแก้ไข
- **C30 (ข้อมูลมากที่สุด)** — เขต 1 ติด C30 เกือบ 150,000 รายการ; แก้ไขผ่าน Appeal 349 (เปิดรอบมกราคม 2569)
- **C998 (ข้ามปี)** — ข้อมูลที่เกินวันที่ 31 ธ.ค. 2567 ต้องทำหนังสือถึงเลขาธิการ สปสช. ชี้แจงความจำเป็น จึงจะขอรับชดเชยได้
- **C750** — ประเภทบริการ OP Type ไม่สอดคล้องกับโมเดลหน่วยบริการ (กทม.) ต้องเลือก OP Type ให้ตรงกับโมเดลเครือข่าย เช่น OP Type 9 = โมเดล 5 เท่านั้น
- **C940/C941** — เงื่อนไขเรียกเก็บ "ผู้ป่วยเปลี่ยนหน่วยบริการ เกิดสิทธิ์ทันที" ไม่ถูกต้อง; C941 คือ ER คุณภาพ ไม่เป็นไปตามเกณฑ์ (ห้ามเบิกค่ายาทางไปรษณีย์, Telemedicine, PP ทุกกรณี, รหัส 55021)
- **C305 (เวลาปิดสิทธิ์ไม่สอดคล้อง)** — ต้องใช้เวลาสิ้นสุดจริงของผู้รับบริการ (เช่น 14:00 น. หากจบตอนบ่าย) ไม่ใช่เวลากวาดข้อมูลอัตโนมัติ (21:00–23:00 น.) — หากติดแล้ว รอรอบอุทธรณ์
- **7 วิธีปิดสิทธิ์** — วิธีที่ 7 คือ Minimal Data Set (6 fields รวม วันที่/เวลาสิ้นสุดบริการ) สำหรับหน่วยบริการสังกัด สป.
- **การชะลอโอนเงิน (Delay Transfer)** — ระบบตรวจสอบอัตโนมัติก่อนโอนวันพุธ 21:00 น.; เขตตรวจ Audit ก่อนปลด; กรณีเข้า Post-Audit ใช้เวลานานขึ้น (มีสิทธิ์อุทธรณ์ 2 ครั้ง) ถ้าไม่ผ่านอาจถูกระงับการให้บริการ
- **Walk-in OP Anywhere** — ยังจ่ายอยู่ สำหรับหน่วยบริการที่ให้บริการผู้มีสิทธิ์ข้ามจังหวัด
- **ยาสมุนไพร** — ประมาณ 30 รายการ (เงื่อนไขปี 68) ชี้แจงเพิ่มเติมวันที่ 30 ต.ค. 2568 ผ่าน Facebook ไลฟ์
- **การโอนเงินตรงไปรพสต.** — ต้องได้รับการตกลงในระดับจังหวัดก่อน แล้วแจ้ง สปสช. ดำเนินการ
