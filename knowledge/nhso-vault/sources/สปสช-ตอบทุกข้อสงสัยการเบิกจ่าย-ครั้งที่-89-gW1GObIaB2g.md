# สปสช. ตอบทุกข้อสงสัยการเบิกจ่าย ครั้งที่ 89

**Channel:** สปสช. | **Duration:** 1:09:10 | **Ingested:** 2026-04-13  
**Aired:** วันศุกร์ที่ 20 กุมภาพันธ์ 2569 เวลา 13:00–14:00 น. (Facebook Live)

## Summary

ครั้งที่ 89 ของรายการ "สปสช. ตอบทุกข้อสงสัยการเบิกจ่าย" ออกอากาศ 20 กุมภาพันธ์ 2569 ประกอบด้วย 3 หัวข้อหลัก คือ (1) การอ่าน Statement และรายงานการโอนเงิน ฉบับมือใหม่ ตอนที่ 2 โดยคุณเดือนเพ็ญ คำมา ผู้เชี่ยวชาญฝ่ายพัฒนาแพลตฟอร์มและนวัตกรรมดิจิทัล ครอบคลุมขั้นตอนดาวน์โหลด Statement UC จากทั้ง eClaim Component และ eClaim Client พร้อมโครงสร้างรายงาน Statement Hub (2) ประกาศ สปสช. 2 ฉบับ คือ การยืนยันสิทธิ์เมื่อสิ้นสุดการรับบริการด้วยระบบ FVS และแนวทางจ่ายค่าใช้จ่ายบริการสาธารณสุข และ (3) กรณีอุทธรณ์ Deny Code IN022 พร้อมการตอบคำถาม Google Form 13 ข้อและคำถาม Facebook Live

วิทยากรพิเศษประจำรายการได้แก่ อาจารย์มานพ โยเฮือง ผู้เชี่ยวชาญฝ่ายตรวจสอบก่อนการจ่ายชดเชย ตอบคำถามเชิงลึกจากหน่วยบริการทั่วประเทศ โดยเฉพาะคำถามเกี่ยวกับรหัสปฏิเสธ IN19, IN23, C998, D001, D415, C349 และกรณีแสดงตนของกลุ่มเปราะบาง สาระสำคัญที่เพิ่มเติมใหม่ในครั้งนี้คือ ประกาศลงวันที่ 20 กุมภาพันธ์ 2569 เรื่องวิธีทางกายภาพ (Physical Verification) สำหรับกลุ่มที่ไม่สามารถยืนยันตัวตนผ่าน FVS หรือ Line ได้ โดยให้ใช้บัตรประชาชนเสียบแทนและเลือกประเภทบริการเป็น "บริการนวัตกรรมโดยการเสียบบัตรประชาชน"

## Key Points

- **Statement UC ดาวน์โหลดได้ 2 ช่องทาง:** eclaim.nhso.go.th/webcomponent และ eClaim Client → เมนูบริการ → รับข้อมูล Statement
- **รอบการออก Statement OP:** 2 รอบ/เดือน — รอบที่ 1 (1–15, ตัดยอด 15, โอน ~25) รอบที่ 2 (16–สิ้นเดือน, ตัดยอดสิ้นเดือน, โอน ~10 เดือนถัดไป)
- **รอบการออก Statement IP:** 1 รอบ/เดือน — โอนเงิน ~วันที่ 14 เดือนถัดไป
- **Statement UC vs Statement Hub:** UC = รายรับของหน่วยบริการเดี่ยว; Hub = มีทั้งพึงรับ (รปสต.) และพึงจ่าย (แม่ข่าย)
- **กรณีอุทธรณ์ IN022/Deny Code IN022:** หน่วยที่ Re-Activate ยังไม่ได้ + ระบบไม่เสถียร + Quota เต็มระหว่างช่วงเปลี่ยนผ่าน สปสช. กำหนดแนวทาง Re-Activate ก่อนบริการ และ FVS/Line ก่อนใช้วิธีอื่น
- **ประกาศใหม่ 20 ก.พ. 2569:** กลุ่มเปราะบาง 4 กลุ่ม (อายุ <15 ปี, ผู้สูงอายุ, ผู้ป่วยมีข้อจำกัดทางร่างกาย, ผู้มีข้อจำกัดเข้าถึงเทคโนโลยี) ยืนยันตัวตนได้ด้วย (1) บัตรประชาชนผู้รับบริการ + Smart Card Reader หรือ (2) บัตรผู้ปกครอง/ผู้ดูแล
- **ประเภทบริการที่ต้องเลือก:** "บริการนวัตกรรมโดยการเสียบบัตรประชาชน (เฉพาะเด็ก ผู้สูงอายุ ผู้มีปัญหาด้านใบหน้า และกลุ่มเปราะบางที่ไม่สามารถยืนยันตัวตนโดยผ่านระบบ FVS/LINE)"
- **IN19 waive อัตโนมัติ:** กรณีเกิดจากระบบ สปสช. เอง → สปสช. ปลด IN19 และ reprocess งวดถัดไป; กรณีหน่วยทำผิด → อุทธรณ์ผ่าน สปสช. เขต/ส่วนกลาง
- **IN23:** เชื่อมฐานวิชาชีพยังไม่แล้วเสร็จ — คลินิกพยาบาล เริ่มเชื่อมได้ 2 ก.พ. 2569; เคสที่ติด IN23 ช่วงก่อนวันดังกล่าว ถือว่าไม่เป็นไปตามประกาศ ไม่มีสิทธิ์อุทธรณ์
- **C998:** ส่งข้อมูลเกิน 31 ธ.ค. ปีงบ → อุทธรณ์ได้โดยแจ้งเหตุผลจำเป็นต่อ สปสช. เขต/ส่วนกลาง (พิจารณารายกรณี)
- **D001 (Deny Code 001):** ข้อมูลบริการไม่เป็นไปตามระยะเวลาที่กำหนด → Auditor ขอตรวจเวชระเบียน → หน่วยบริการต้องเข้าระบบอุทธรณ์ภายใน 10 วัน
- **D415:** รหัส D นำหน้ารหัส C เดิม — D415 = C415 = เบิก Telemedicine แต่ไม่มีศักยภาพขึ้นทะเบียน; ให้ดูความหมายจาก C เดิม
- **C349:** ไม่ดำเนินการ Attend (ปิดสิทธิ์) เลย → ไม่มีการอุทธรณ์ในระบบ Appeal 349
- **OPBKK รหัส Statement 99:** กรณีรับส่งต่อนอกเครือข่าย เขต 13 → Statement ชั่วคราว รหัส 99 → สปสช. จะออก Statement จริงให้อีกรอบ หน่วยบริการไม่ต้องดำเนินการใดๆ
- **NHO Digital Platform:** มีปัญหาการบันทึก Service Code แผนไทย → แจ้ง Provider Contact Center 1330 กด 5 กด 1
