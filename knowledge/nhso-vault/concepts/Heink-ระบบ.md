# Heink (Health Information Exchange)

## Definition

**Heink** คือระบบแลกเปลี่ยนข้อมูลสุขภาพ (Health Information Exchange) ที่พัฒนาโดย **BDI (สถาบันข้อมูลขนาดใหญ่ / Big Data Institute)** ซึ่งเป็นองค์กรภายใต้ สวทช. (สำนักงานพัฒนาวิทยาศาสตร์และเทคโนโลยีแห่งชาติ) ทำหน้าที่รวมศูนย์ประวัติการรักษาของประชาชนจากหน่วยบริการต่างๆ ให้ผู้ให้บริการที่มีสิทธิ์เข้าถึงได้แบบ Real-time

## วัตถุประสงค์

- ให้บุคลากรทางการแพทย์ดูประวัติการรักษาเก่าของผู้ป่วยได้ทันที โดยไม่ต้องรอให้ผู้ป่วยเล่าประวัติเอง
- ลดการสั่งยาซ้ำซ้อนหรือตรวจซ้ำ
- เชื่อมข้อมูลระหว่างหน่วยบริการปฐมภูมิ (คลินิกนวัตกรรม) กับโรงพยาบาล
- รองรับกฎหมาย PDPA โดยมีการยืนยันตัวตนผู้เข้าถึงข้อมูลผ่านสภาวิชาชีพ

## เหตุผลที่ต้องเชื่อมกับ OSS และสภาวิชาชีพ (PDPA)

ประวัติการรักษาเป็น **Sensitive Data** ตามกฎหมาย PDPA → สปสช. และ BDI ต้องรู้ว่าใครเป็นผู้เข้าถึงข้อมูล ดังนั้น:
- ผู้ให้บริการต้อง Login ด้วย OSS credential ที่เชื่อมกับสภาวิชาชีพ (เลขบัตรประชาชน + เลข ว.)
- ระบบตรวจสอบว่าเป็นวิชาชีพที่ได้รับอนุญาตก่อนแสดงข้อมูล
- การ reactivate ใน OSS คือเงื่อนไขเบื้องต้นก่อนใช้ Heink ได้

## ผู้พัฒนา

| องค์กร | บทบาท |
|---|---|
| BDI (สถาบันข้อมูลขนาดใหญ่) | พัฒนาและดูแล Heink |
| สวทช. | ต้นสังกัดของ BDI |
| สปสช. | เชื่อมระบบ OSS เพื่อยืนยันตัวตนก่อนเข้า Heink |
| สภาวิชาชีพ | ยืนยันสถานะใบประกอบวิชาชีพผ่าน API |

## ขั้นตอนการเข้าใช้ Heink

1. ผู้ให้บริการ login ด้วย credential จาก OSS (เชื่อมสภาวิชาชีพแล้ว)
2. ระบุเลขบัตรประชาชนผู้ป่วย
3. Heink แสดงประวัติการรักษาจากหน่วยบริการที่ผู้ป่วยเคยใช้บริการ
4. ผู้ให้บริการซักประวัติประกอบกับข้อมูลเก่าได้ทันที
5. บันทึกการรักษาในครั้งนี้จะถูกรวมเข้า Heink อัตโนมัติ

## ระบบที่ทำงานร่วมกัน

- **OSS (One Stop Service)** — ยืนยันตัวตนผู้ให้บริการ
- **AMT** — ซอฟต์แวร์หน่วยบริการที่ integrate กับ Heink
- **New AEN** — ซอฟต์แวร์คลินิกที่ integrate กับ Heink
- **เป๋าตังค์ (KTB)** — ประชาชนดูประวัติตัวเองผ่านฝั่งประชาชน (คนละช่องทาง)
