# Financial Data Hub (FDH) และ NHO Digital Platform

## Definition

Financial Data Hub (FDH) คือระบบกลางของกระทรวงสาธารณสุขที่รวบรวมข้อมูลธุรกรรมการให้บริการของหน่วยบริการสังกัดกระทรวง ก่อนส่งต่อให้ สปสช. ผ่าน NHO Digital Platform (EClaim)

## How It Works

### สายการส่งข้อมูล (หน่วยกระทรวง)
```
หน่วยบริการ → Financial Data Hub (FDH) → NHO Digital Platform (EClaim) → ประมวลผลจ่าย
```

1. **Financial Data Hub:** รับข้อมูลทุกธุรกรรมของหน่วย ทั้งที่ประสงค์เบิกและไม่ประสงค์เบิก
2. **NHO Digital Platform / EClaim:** ตรวจสอบและประทับสถานะ C (รหัสปัญหา) ทุกรายการที่ผ่านเข้ามา

### สถานะรหัส C ใน EClaim
- รหัส C ที่ประทับแล้วใน EClaim **ไม่สามารถลบออกได้** เนื่องจากเป็น Digital Fingerprint / หลักฐานการส่งข้อมูล
- ถือว่าทุกรายการที่ส่งเบิกเข้ามา คือการ "ตั้งให้ สปสช. เป็นลูกหนี้" กับหน่วยบริการ
- **แหล่งอ้างอิงหลัก:** ให้ยึดสถานะล่าสุดใน **IEP (Individual Electronic Payment)** ของ EClaim

### กรณีไม่ประสงค์เบิก (UCC = 2)
- หากหน่วยบริการส่งข้อมูลเข้า FDH แต่ไม่ประสงค์เบิกกับ สปสช. ให้ปรับฟิลด์ `UCC = 2` ในชุดข้อมูล 16 แฟ้มมาตรฐาน EClaim
- ส่งเข้า FDH อีกครั้ง → FDH จะ refresh ข้อมูล
- EClaim จะประมวลผลจ่ายเป็น 0 บาท (ไม่จ่าย) และแสดงสถานะนั้น
- รายการใน EClaim ยังคงแสดงอยู่ (ไม่หายไป) แต่สถานะจะเป็น "ไม่ประสงค์เบิก"

### รหัส C179 (ข้อมูลซ้ำ)
เกิดจาก:
1. บันทึกข้อมูลซ้ำกับรายการที่มีอยู่แล้วหรือผ่านพร้อมจ่ายแล้ว
2. ผู้ป่วยใน Admit วันเดียวกันมากกว่า 1 ครั้งในหน่วยบริการเดียว

วิธีแก้ไข:
- กรณีซ้ำ: ไม่ต้องแก้ไข รายการอื่นที่เป็นรายการเดิมได้รับการจ่ายแล้ว
- หากต้องการแก้: ขยับวันที่รับบริการในเรคคอร์ดที่ติด C179 เป็นวันอื่น แล้วเลือก UCC=2

### รหัส C950 (แฟ้มข้อมูลไม่ครบ)
- ข้อมูล 16 แฟ้มมาตรฐาน EClaim ส่งมาไม่ครบถ้วน
- ตรวจสอบแฟ้มที่เกี่ยวข้อง: OPD, สิทธิ์, ค่าใช้จ่าย, ADP (Item)

### หน่วยบริการรหัส 9 หลัก
- หน่วยนวัตกรรมใหม่อาจใช้รหัส 9 หลัก
- ระบบ Simulate for DMIS รองรับทั้ง 5 หลักและ 9 หลักสำหรับรายงานการจ่าย
