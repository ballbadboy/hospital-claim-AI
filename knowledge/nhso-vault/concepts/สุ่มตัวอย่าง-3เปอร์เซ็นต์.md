# Random Sampling 3% (การสุ่มตรวจเวชระเบียน)

## Definition

Random Sampling 3% คือกระบวนการสุ่มเลือกเวชระเบียนผู้ป่วยในอย่างน้อย 3% จากทุกหน่วยบริการทั่วประเทศที่รับเงิน IP มาตรวจสอบโดย Auditor เพื่อความถูกต้องของการลงรหัสโรค DRG และ AdjRW ที่ใช้เรียกเบิก

## How It Works

1. **สุ่มตัวอย่าง:** สปสช.สุ่มเลือกเวชระเบียนอย่างน้อย 3% จากทุกหน่วยบริการ (หน่วยบริการ 1,000+ แห่ง)
2. **ตัดข้อมูลที่อุทธรณ์:** ข้อมูลที่หน่วยบริการยังอุทธรณ์อยู่จะไม่ถูกนำมาใช้ในการตรวจสอบครั้งนี้
3. **Auditor ตรวจสอบ:** ทีม Auditor ตรวจสอบว่าการลงโค้ดโรค ความถูกต้องของบริการ และ AdjRW สอดคล้องกับบันทึกทางการแพทย์
4. **ปรับ AdjRW:** หากพบว่า AdjRW สูงเกินจริง → ปรับลด (หักเงินคืน) / หากต่ำเกินจริง → ปรับเพิ่ม (จ่ายเพิ่ม)
5. **สรุปผล:** ผลรวมของการปรับปรุงทั้งหมดใน 3% แสดงเป็นยอดเดียวใน **คอลัมน์ 5 ของ Statement**
6. **ผ่านกระบวนการอุทธรณ์แล้ว:** ผลที่ปรากฏใน Statement คอลัมน์ 5 ถือว่าสิ้นสุดกระบวนการอุทธรณ์

## ตัวอย่าง รพ.พุทธชินราช

- สุ่มตรวจประมาณ 385 รายการ (จากเวชระเบียนทั้งหมด ~12,000+ ราย)
- ผลการตรวจ: ต้องคืนเงิน 44,954 บาท (คอลัมน์ L ใน Excel สรุป)
- ดูรายละเอียดได้จาก Statement รหัส `6809_IP_03`

## ตัวอย่าง รพ.ยุพราชสระแก้ว

- ผลการสุ่มตรวจ 3%: ต้องคืนเงิน 220,000 บาท

## การดูรายละเอียด

เข้าระบบ EK Web หรือ EKB Component แล้วดาวน์โหลด Statement ที่มีรหัส:
- `6809_IP_03` — ผลการสุ่มตรวจ (Random Sampling)

ในไฟล์ Statement จะแสดง:
- ชีทสรุป: ยอดรวม IP หักคืน/จ่ายเพิ่ม
- ชีทรายบุคคล: แต่ละเคสถูกปรับอะไร เท่าไหร่
- ฟิลด์ Trans ID: ใช้ติดตามรายการเฉพาะ

## ข้อจำกัด

- Statement ปัจจุบันแสดงเฉพาะ AdjRW ที่ผ่านการตรวจสอบแล้ว ไม่แสดง AdjRW เดิมเปรียบเทียบ (สปสช.มีแผนปรับปรุงในอนาคต)
- ต้องดาวน์โหลดไฟล์ IEP เพิ่มเติมเพื่อเปรียบเทียบ AdjRW เก่าและใหม่
