# Deny Code IN022 — อุทธรณ์กรณีปิดสิทธิ์ไม่สมบูรณ์ช่วงเปลี่ยนผ่าน

## Definition

**IN022** (หรือ IN22) คือรหัสปฏิเสธที่ระบุว่า **ไม่ได้ปิดสิทธิ์ผ่านช่องทางที่กำหนด (FVS หรือ LINE Application)** เกิดขึ้นกับหน่วยบริการนวัตกรรมในช่วงเปลี่ยนผ่านระบบยืนยันตัวตนใหม่

## สาเหตุหลักของ IN022

| สาเหตุ | คำอธิบาย |
|--------|----------|
| Re-Activate ยังไม่ได้ดำเนินการ | หน่วยยังไม่เชื่อมฐานวิชาชีพกับ สปสช. |
| ระบบไม่เสถียร | ระบบ FVS/Line มีปัญหาช่วงเปิดใช้งานใหม่ |
| ไม่ปิดสิทธิ์ผ่าน FVS/Line | ไม่ดำเนินการผ่านช่องทางที่กำหนด |
| Quota เต็ม | โควต้ารายวันหมดในช่วงเวลาสั้น ยืนยันตัวตนไม่ได้ |

## แนวทางการพิจารณาอุทธรณ์

สปสช. พิจารณา **ไม่เข้าเกณฑ์พิจารณา** หากหน่วยบริการ:
- ยังไม่ได้ Re-Activate
- ไม่มีการบันทึก IN022 จากความพยายามปิดสิทธิ์ด้วย FVS/Line
- ใช้ช่องทาง Dip Chip บัตรประชาชนแทน FVS/Line โดยไม่ผ่านขั้นตอนที่กำหนด

**หน่วยต้องดำเนินการก่อน:**
1. Re-Activate ใน OSS ให้แล้วเสร็จ
2. ผู้รับบริการยืนยันตัวตนผ่าน **FVS หรือ LINE Application** (ลอง FVS/Line ก่อนเสมอ)
3. หากทำไม่ได้ → แจ้งเหตุผลรายกรณีประกอบอุทธรณ์

## ความแตกต่าง IN022 vs IN19

| รหัส | ความหมาย | การแก้ไข |
|------|----------|----------|
| **IN022** | ไม่ปิดสิทธิ์ผ่าน FVS/Line เลย | Re-Activate + ใช้ FVS/Line ก่อน |
| **IN19** | เลือกวิธียืนยันตัวตนไม่ตรงประเภทบริการ | สปสช. waive อัตโนมัติ หรืออุทธรณ์ |

## กรณี Quota เต็ม

หากโควต้าเต็มทำให้ไม่สามารถให้บริการได้ในช่วงเวลาดังกล่าว → ต้องชี้แจงเหตุผลในรายงานอุทธรณ์

## ช่องทางอุทธรณ์

- แจ้ง **สปสช. เขต** หรือ **สปสช. ส่วนกลาง**
- แนบรายละเอียด: ชื่อหน่วย, รหัสหน่วย, วันที่บริการ, จำนวนรายการ, เหตุผล
- อ้างอิงประกาศจ่ายค่าใช้จ่ายมีผลบังคับใช้ 12 มกราคม 2569
