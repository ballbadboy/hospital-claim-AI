# Appeal 349 — ระบบอุทธรณ์ข้อมูล C30

## Definition

Appeal 349 คือ **เว็บแอปพลิเคชันสำหรับแก้ไขข้อมูลที่ติด C30** (รหัสปฏิเสธทั่วไป) โดยเฉพาะกรณีที่มีปริมาณมาก ซึ่ง สปสช. เปิดให้หน่วยบริการส่งข้อมูลแก้ไขเป็นรอบๆ

## กำหนดการเปิดใช้งาน

- **ปีงบประมาณ 2568:** เปิดรอบ **มกราคม 2569** (เหมือนกับปีที่ผ่านมา)
- สปสช. จะประกาศรอบที่ชัดเจนอีกครั้ง

## ขั้นตอนการใช้งาน

1. ตรวจสอบ C30 ของหน่วยบริการผ่าน [[concepts/Dashboard-CD9]]
2. เรียงตามค่าใช้จ่ายเรียกเก็บ — แก้ไขรายการที่สำคัญก่อน
3. เมื่อสปสช. เปิดรอบ → เข้า Appeal 349 และส่งข้อมูลที่แก้ไขแล้ว
4. ระบบจะนำข้อมูลเข้าประมวลผลใหม่

## C-Code ที่ใช้ Appeal 349

- **C30** เป็นหลัก — ข้อมูลไม่ผ่านการตรวจสอบทั่วไป
- C ตัวอื่นๆ อาจมีช่องทางแก้ไขผ่านระบบอื่น (ดู [[concepts/c-code-การแก้ไข]])

## ข้อจำกัด

- เปิดเป็นรอบ ไม่ได้เปิดตลอด
- ใช้สำหรับข้อมูลในปีงบประมาณ 2568 เท่านั้น
- ข้อมูลข้ามปี (C998) ต้องใช้กระบวนการทำหนังสือแยกต่างหาก

## รายละเอียดรอบอุทธรณ์ C305 — กุมภาพันธ์ 2569

จากไลฟ์ครั้งที่ 86 (30 ม.ค. 2569):

- **URL:** ap349.nhso.go.th
- **ระยะเวลา:** 1–28 กุมภาพันธ์ 2569
- **ขอบเขต:** C305 ทั้งหมดปีงบ 2568 (~800,000 records ที่ไม่ใช่ระบบผิดพลาด)

### เงื่อนไขการอุทธรณ์ C305

1. ดาวน์โหลดรายการ C305 ของหน่วยจากเว็บไซต์ สปสช. (แยกรายหน่วย)
2. ใส่เหตุผลอุทธรณ์รายแถว (row-by-row)
3. แนบเอกสารประกอบ **แยกรายบุคคล** (ไม่ใช่ชุดรวม) — เช่น เอกสาร ฮีโมชีส กรณีฟอกไต
4. มีลายมือผู้มีอำนาจลงนามรับรอง
5. อัปโหลดตามโครงสร้างที่ สปสช. กำหนด
6. ระยะเวลาตรวจสอบ **30 วัน** (มาตรฐาน)
7. หลังอนุมัติ → ออก Statement ปกติ

### ช่องทางอุทธรณ์แยกตามประเภทบริการ

| ประเภท | ช่องทาง |
|--------|---------|
| C305 ทั่วไป | ap349.nhso.go.th |
| กรณีไตวาย/AVF/AVG | ระบบ eClaim → EPL (แยกต่างหาก) |
| IN22 (PP ใน Krungthai DHP) | สปสช. waive + reprocess อัตโนมัติ |
| C30 (eClaim เวลาคลาดเคลื่อน) | ap349.nhso.go.th เช่นกัน |

### ปีงบประมาณ 2569 (ตั้งแต่ 1 ม.ค. 2569)

ไม่มีการติด C305 แล้ว ระบบแจ้ง error ทันทีตอนส่งข้อมูล หน่วยบริการสามารถแก้ไขได้เลย

### วิธีแก้สถานะ 7 — Tran ID Format ผิด

สาเหตุ: เมื่อ copy Tran ID จากระบบ แล้ว paste ลงใน Excel แบบปกติ Excel จะแปลงตัวเลขยาวเป็น Scientific Notation หรือตัวเลขซ้ำกัน ทำให้ระบบไม่สามารถจับคู่ได้

**วิธีแก้:**
1. ดาวน์โหลด template CSV จากหน้าบันทึกอุทธรณ์
2. เปิดไฟล์ CSV — จะเห็น Tran ID ในรูปแบบที่อ่านได้ (ขึ้นต้น 6 หรือ 7 ตามด้วยเดือน เช่น `6708xxxxxxx`)
3. เมื่อ copy Tran ID มาวางใน template ให้ใช้ **วางค่า (Paste as Values)** เท่านั้น อย่า paste ปกติ
4. คลิกที่เซลล์แล้วตรวจสอบว่าค่าที่แสดงไม่ซ้ำกันทุก record
5. เพิ่มคอลัมน์ **reson (เหตุผล)** ทุก record — ห้ามเว้นว่าง
