# OSS Two-Factor Authentication (OSS-2FA)

## Definition

**OSS-2FA** คือระบบยืนยันตัวตน **สองชั้น (Two-Factor Authentication)** สำหรับการเข้าใช้งานระบบต่างๆ ของ สปสช. ผ่านแพลตฟอร์ม **One Stop Service (OSS)** โดยผสมผสาน:
- ชั้นที่ 1: **Username + Password** บน OSS
- ชั้นที่ 2: **ThaiD** (แอปบัตรประชาชนอิเล็กทรอนิกส์ของกรมการปกครอง) — สแกน QR Code

เปิดใช้งานจริงตั้งแต่วันที่ **21 มีนาคม 2569** (เปลี่ยนแบบกะทันหัน ทำให้หน่วยบริการหลายแห่งเข้าระบบไม่ได้ในช่วงสุดสัปดาห์แรก)

## ระบบเก่า vs. ระบบใหม่

| รายการ | Data Center (ระบบเก่า) | OSS + 2FA (ระบบใหม่) |
|--------|------------------------|----------------------|
| Username | ตัวเลข 12–13 หลัก | ตั้งเองได้ (อักษร+ตัวเลข) |
| Factor | Single (User + Pass) | Two (User + Pass + ThaiD) |
| ความปลอดภัย | ต่ำ | สูง |
| เข้าระบบผ่าน | ตรงๆ | หน้า OSS ทุกครั้ง |

## How It Works — ขั้นตอนการใช้งาน

### การเตรียมครั้งแรก
1. **ดาวน์โหลด ThaiD** — Google Play หรือ App Store (แอปของกรมการปกครอง)
2. **ลงทะเบียน ThaiD** — ถ่ายรูปบัตรประชาชน + สแกนใบหน้า + ตั้ง Passcode 7 หลัก
   - ลงทะเบียนได้ด้วยตัวเองออนไลน์ หรือไปที่หน่วยราชการ (สบท.)
3. **ตรวจสอบ PID** — เลขบัตรประชาชนที่ใช้ใน ThaiD ต้องตรงกับที่ผูกไว้บน OSS
4. **ตรวจสอบบทบาท (Role)** ใน OSS — เฉพาะผู้มีอำนาจลงนาม/ผู้รับมอบอำนาจ สร้าง User และกำหนด Role ได้
5. **สร้าง User ใหม่บน OSS** หากยังใช้ User ของ Data Center อยู่

### การเข้าระบบทุกครั้ง
1. ไปที่หน้าโปรแกรมที่ต้องการ (จะถูก redirect ไปหน้า OSS อัตโนมัติ)
2. กด **Login ด้วย ThaiD**
3. เปิดแอป ThaiD บนมือถือ → **สแกน QR Code** บนหน้าจอคอมพิวเตอร์
4. เข้าสู่ระบบ → เลือกเมนูตามบทบาทที่มี

### การสลับโปรไฟล์ (กรณีมี 2 หน่วยบริการ)
- เข้าหน้า OSS → เลือก "โปรไฟล์อื่น" → คลิกเลือกโปรไฟล์ที่ต้องการ → ออกจากระบบ → เข้าระบบปลายทางใหม่

## ข้อควรระวัง

| ข้อควรระวัง | รายละเอียด |
|------------|-----------|
| **1 ThaiD = 1 มือถือ** | ห้ามให้คนอื่นสแกน QR แทน — ระบบเก็บ log ชื่อผู้สแกน ถ้าให้คนอื่นสแกน = ความรับผิดชอบตกที่เราเหมือนให้บัตรประชาชนแก่ผู้อื่น |
| **อย่าสแกน 2 เครื่องพร้อมกัน** | จะถูกตัดทั้ง 2 เครื่อง อาจ lock ออกไม่ได้ |
| **ออกจากระบบทุกครั้ง** | Session ค้างทำให้ระบบมีปัญหา ออก (Log out) ก่อนเข้าระบบอื่น |
| **1 PID ต่อ 1 H-Code** | ถ้าต้องการเข้า 2 หน่วยด้วย User เดียว ต้องสลับโปรไฟล์ หรือแยก PID ต่อ H-Code |

## ระบบที่ใช้ OSS-2FA แล้ว (ณ มี.ค. 2569)

- ระบบ reactivate หน่วยนวัตกรรม
- ระบบปิดสิทธิ์ / บริการนวัตกรรม
- ระบบ CKD DMIS ([[concepts/CKD-ไตวายเรื้อรัง-DMIS]])
- Seamless for DMIS ([[concepts/Seamless-DMIS]])
- ระบบ NHAP / NAP และระบบอื่นๆ ของ สปสช. (ทยอยปรับ)

## สาเหตุที่ FVS/ThaiD ช้า

ช่วงแรกของการเปิดใช้งาน ระบบ ThaiD ของกรมการปกครองอาจ **rate limit** เนื่องจากมีผู้ใช้งานพร้อมกันมาก อยู่ระหว่างเจรจาเพิ่ม quota กับกรมการปกครอง
