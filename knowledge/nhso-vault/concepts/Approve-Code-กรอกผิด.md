# ข้อผิดพลาดในการบันทึก Approve Code (AEN Code)

## Definition

Approve Code (หรือ AEN Code / App Code / AENT Code) คือรหัสตัวเลขที่ระบบ New AEN ออกให้เมื่อหน่วยบริการทำการปิดสิทธิ์ผู้ป่วยเรียบร้อยแล้ว หน่วยบริการต้องนำรหัสนี้ไปบันทึกในระบบ e-Claim (หน้า F1) เพื่อยืนยันการปิดสิทธิ์

## ปัญหาที่พบบ่อย: ช่องว่างติดมา

### สาเหตุ

เมื่อคัดลอก (Copy) Approve Code จากหน้าเว็บแอปพลิเคชัน New AEN แล้วมาวาง (Paste) ในช่อง AENT Code ของระบบ e-Claim อาจมี **ช่องว่าง (space)** ติดมาด้วย ทำให้รหัสที่บันทึกไม่ตรงกับที่ระบบออกให้

### ผลที่เกิดขึ้น

- ข้อมูลไม่ผ่านการตรวจสอบ → ติด C-code (เช่น C30, C305)
- ระบบตรวจสอบพบว่า Approve Code ไม่ถูกต้อง

## วิธีแก้ไข

1. **ตรวจสอบช่อง AENT Code** ในระบบ e-Claim หน้า F1
2. **ลบช่องว่าง** ทั้งหน้าและหลังรหัสออกให้หมด
3. **ตรวจสอบความครบถ้วน** ของตัวเลข (ไม่ขาด ไม่เกิน)
4. **ส่งข้อมูลใหม่** อีกครั้ง

## ระบบที่เกี่ยวข้อง

| ระบบ | บทบาท |
|------|-------|
| New AEN (New Or) | ออก Approve Code / AEN Code หลังปิดสิทธิ์ |
| e-Claim (ECIM) | รับบันทึก AENT Code ในหน้า F1 |

## ฟังก์ชันการปิดสิทธิ์ใน New AEN

ระบบ New AEN มี 2 ฟังก์ชัน:
1. **ใช้บัตรประชาชนตัวเอง** — ผู้ป่วยถือบัตรมาเอง
2. **ใช้บัตรประชาชนของผู้รับรอง** — กรณีกลุ่มเปราะบาง
3. **บัตรประชาชนเสมือนผ่าน ThaiD** — แอปพลิเคชัน ThaiD
