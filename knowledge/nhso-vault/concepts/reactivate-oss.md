# Reactivate หน่วยบริการ ใน OSS

## Definition

**Reactivate** คือกระบวนการที่หน่วยบริการนวัตกรรมต้องดำเนินการใน **ระบบ One Stop Service (OSS)** ของ สปสช. เพื่อยืนยันความพร้อมให้บริการในระบบใหม่ปีงบ 2569 โดยต้องอัปเดตข้อมูลสำคัญ ได้แก่ พิกัด GPS, บุคลากร, และเชื่อมกับสภาวิชาชีพ ก่อนเปิดให้บริการวันที่ **12 มกราคม 2569**

ทดสอบระบบ Line NHSO: **7 มกราคม 2569**

## ใครต้องทำ

**ทุกหน่วยบริการนวัตกรรม** ที่ต้องการให้บริการในระบบสปสช.ต่อไป — ทำผ่านเว็บ One Stop Service (ลิงก์แจ้งในแชท Zoom และ Facebook Live วันประชุม)

## ผู้ที่มีสิทธิ์ทำ Reactivate

- **ผู้ประกอบกิจการ** (เจ้าของคลินิก) หรือ
- **ผู้ได้รับมอบอำนาจ** ตามที่กรมพัฒนาธุรกิจการค้ากำหนด

## ขั้นตอน Reactivate

### 1. ตรวจสอบที่อยู่ที่ตั้ง
- เข้าเมนู "ข้อมูลทั่วไป" → ตรวจสอบที่อยู่ว่าถูกต้องหรือไม่

### 2. อัปเดตพิกัด GPS (สำคัญที่สุด)
- กด "ตำแหน่งปัจจุบัน" → ระบบดึง Latitude/Longitude อัตโนมัติ
- **ต้องทำที่คลินิกเท่านั้น** — ห้ามทำจากที่บ้าน เพราะพิกัดจะผิด ทำให้สแกน Face/Line ไม่ได้ (ระยะทางจากพิกัดในระบบเกินกำหนด)

### 3. บันทึกบุคลากร (Full-time และ Part-time)
- ระบุ **เลขบัตรประชาชน** และ **เลข ว.** (เลขใบประกอบวิชาชีพ) ของทุกคน
- ระบุวัน-เวลาปฏิบัติงาน (ใช้คำนวณ Workload ต่อหน่วยบริการ)
- ข้อมูลนี้ต้องตรงกับที่แจ้งไว้ใน **สพ.6** (ใบอนุญาตสถานพยาบาล)
- ระบบคำนวณ man-hour: เช่น แพทย์ทำงาน 08:00-12:00 = 4 man-hours → ให้บริการได้กี่คน (ตามมาตรฐานสภาวิชาชีพ)

### 4. เชื่อมกับสภาวิชาชีพ (API)
- ระบบ OSS จะ call API ไปยังสภาวิชาชีพเพื่อยืนยันว่า เลข ว. นั้นถูกต้องและยังมีใบอนุญาต
- **ถ้าเชื่อมสำเร็จ:** ปุ่มสีเขียว = Reactivate เรียบร้อย
- **ถ้าเชื่อมไม่ได้:** ปุ่มสีแดง = ต้องติดต่อสภาวิชาชีพให้เชื่อม API กับสปสช.ก่อน

### 5. แอด Line NHSO
- หลัง reactivate สำเร็จ → แอด Line NHSO (@NHO) เป็นเพื่อน
- ระบบจะแสดง Countdown ถึงวันเปิดระบบ (7 ม.ค. 69 ทดสอบ, 12 ม.ค. 69 เปิดจริง)
- กดเมนู "30 บาท หน่วยนวัตกรรม" → ดูโควต้า, สร้าง QR ให้ประชาชนสแกน

## สถานะสภาวิชาชีพ (ณ 30 ธ.ค. 68)

| สภา | สถานะ |
|---|---|
| แพทยสภา | เชื่อม API เรียบร้อย |
| ทันตแพทยสภา | เชื่อม API เรียบร้อย |
| สภาเภสัชกรรม | เชื่อม API เรียบร้อย |
| สภากายภาพบำบัด | เชื่อม API เรียบร้อย |
| สภาเทคนิคการแพทย์ | เชื่อม API เรียบร้อย |
| สภาการแพทย์แผนไทย | เชื่อม API เรียบร้อย |
| สภาการพยาบาล | **ยังดำเนินการอยู่** — reactivate ยังไม่สมบูรณ์ |

## ความสัมพันธ์กับ สพ.6

สพ.6 = เอกสารใบอนุญาตสถานพยาบาล (กรมสนับสนุนบริการสุขภาพ) ที่ระบุรายชื่อบุคลากรทางการแพทย์ที่ปฏิบัติงาน ต้องตรงกับที่บันทึกใน OSS ของสปสช.เสมอ เพราะสปสช.ใช้ข้อมูล สพ.6 ในการ audit

## CPP (โปรแกรมข้อมูลพื้นฐานและศักยภาพ)

- โปรแกรม CPP เวอร์ชันใหม่เปิดตัว 30 ธ.ค. 68 พร้อมกัน
- **เวอร์ชันเก่าใช้ไม่ได้อีกต่อไป** นับจาก 31 ธ.ค. 68
- เวอร์ชันใหม่รองรับการบันทึกบุคลากรพยาบาลและกายภาพรายบุคคล (เวอร์ชันเก่าไม่มี)
- หน่วยที่เคยบันทึกใน CPP เวอร์ชันเก่าต้องบันทึกใหม่ทั้งหมดในเวอร์ชันใหม่

## เหตุผลที่ต้องยืนยันตัวตนวิชาชีพ (PDPA)

ระบบใหม่จะให้บุคลากรเข้าดูประวัติการรักษาของคนไข้ผ่าน Heink ซึ่งเป็น sensitive data ตามกฎหมาย PDPA — สปสช.จึงต้องรู้ว่าใครเป็นคนเข้าถึงข้อมูล จึงต้องเชื่อมกับสภาวิชาชีพเพื่อยืนยัน

## ข้อมูลเพิ่มเติมจากปี 2569 (คลินิกพยาบาล)

จากการประชุม 27 กุมภาพันธ์ 2569 สำหรับคลินิกพยาบาลโดยเฉพาะ:
- พยาบาลต้องลงทะเบียนประเภท **4.1** (วิชาชีพทั่วไป) หรือ **4.2.26** (เวชปฏิบัติ) ให้ถูกต้อง
- Part-time: 1 คนไม่สามารถซ้อนทับเวลาให้บริการในคลินิกมากกว่า 1 แห่งพร้อมกัน
- ข้อมูลในระบบ reactivate ต้องสอดคล้องกับ **สพ. 7** (ใบอนุญาตประกอบกิจการ)

## กติกาใหม่ Reactivate (มีผลตั้งแต่ 17 ม.ค. 2569)

**ก่อนหน้า:** Reactivate มีผลทันทีในวันที่ทำ

**กติกาใหม่:** Reactivate จะ**มีผลวันที่ 1 ของเดือนถัดไป** เท่านั้น
- ทำ reactivate เมื่อไหรก็ได้ แต่จะยังให้บริการไม่ได้จนกว่าจะถึงวันที่ 1 ของเดือนถัดไป
- เหตุผล: เพื่อให้ สปสช. บริหารโควต้ารายสัปดาห์ได้มีเสถียรภาพ ไม่มีหน่วยใหม่เข้าระบบกลางสัปดาห์

**กรณีเปลี่ยนผู้ประกอบกิจการ:** ต้อง reactivate ใหม่ทั้งหมด และรอถึงวันที่ 1 ของเดือนถัดไปก่อนจึงจะให้บริการได้ ไม่สามารถให้บริการในรูปแบบเดิมระหว่างนั้นได้
