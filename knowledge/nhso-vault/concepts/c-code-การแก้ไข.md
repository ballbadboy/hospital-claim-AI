# C-Code — คู่มือแก้ไขรหัสปฏิเสธ

## Definition

C-Code คือรหัสที่ระบบ สปสช. ใช้แจ้งเหตุผลการปฏิเสธข้อมูลการเบิกจ่าย หน่วยบริการต้องแก้ไขข้อมูลให้ถูกต้องและส่งใหม่ภายในระยะเวลาที่กำหนด (โดยทั่วไปภายในปีงบประมาณ)

## ระยะเวลาแก้ไข

- **ภายในปีงบประมาณ 2568:** เหลือประมาณ 70 วัน (นับจากวันที่ 24 ต.ค. 2568)
- **รอบ Re-run ผ่าน Appeal 349:** เดือนมกราคม 2569 (สำหรับ C30 เป็นหลัก)
- **ข้ามปี (C998):** ต้องทำหนังสือถึงเลขาธิการ สปสช.

## ตารางรหัส C-Code ที่พบบ่อย

| รหัส | ความหมาย | วิธีแก้ไข |
|------|-----------|-----------|
| **C30** | ข้อมูลไม่ผ่านการตรวจสอบทั่วไป | แก้ไขข้อมูลผ่าน Appeal 349 (เปิด ม.ค. 2569) |
| **C38 / C438** | สิทธิ์ไม่ถูกต้อง (รวมกรณีระบบล่ม) | รอสปสช. ประมวลใหม่ หรือตรวจสอบสิทธิ์ใหม่ |
| **C305** | เวลาปิดสิทธิ์ไม่สอดคล้องกับเวลาบริการ | ใช้เวลาสิ้นสุดบริการจริง (ดู [[concepts/การปิดสิทธิ์]]) |
| **C350** | ปิดสิทธิ์ผิดเวลา (ชื่อเต็ม) | รอรอบอุทธรณ์ |
| **C370** | บริการในเคลมไม่ใช่บริการปฐมภูมิ / บริการข้ามจังหวัดไม่ถูกต้อง | ตรวจสอบรหัสโครงการ Walk-in |
| **C750** | OP Type ไม่ตรงกับโมเดลหน่วยบริการ (กทม.) | เลือก OP Type ให้ตรงโมเดล เช่น OP Type 9 = โมเดล 5 |
| **C940** | เงื่อนไขเรียกเก็บ "ผู้ป่วยเปลี่ยนหน่วย เกิดสิทธิ์ทันที" ไม่ถูกต้อง | แก้เงื่อนไขใน e-Claim และเบิกเฉพาะ Free Schedule |
| **C941** | ER คุณภาพไม่เป็นไปตามเกณฑ์ | ลบรายการที่ห้ามเบิก: ยาทางไปรษณีย์, Telemedicine, PP, รหัส 55021 |
| **C998** | ข้อมูลเกินวันที่ 31 ธ.ค. ของปีงบ (ข้ามปี) | ทำหนังสือชี้แจงเหตุผลถึงเลขาธิการ สปสช. |

เคสที่เคยเห็นสิทธิ์ว่างแล้วเลือก "ผู้ป่วยเปลี่ยนหน่วย เกิดสิทธิ์ทันที" ในหน้า e-Claim → ติด C940
