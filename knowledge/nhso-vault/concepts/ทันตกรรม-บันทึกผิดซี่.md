# ทันตกรรม — บันทึกข้อมูลผิดซี่ฟัน

## Definition

ปัญหาที่เกิดขึ้นเมื่อคลินิกทันตกรรมอื่นบันทึกหัตถการฟัน (เช่น ถอนฟัน) ผิดซี่ในระบบ สปสช. แล้วคลินิกใหม่ที่ผู้ป่วยมาใช้บริการต้องการบันทึกหัตถการ (เช่น อุดฟัน) ในซี่เดียวกัน แต่ระบบปฏิเสธเพราะฐานข้อมูลระบุว่าซี่นั้น "ถอนไปแล้ว"

## ตัวอย่างสถานการณ์

- คลินิก A บันทึกถอนฟันซี่ 16 (ผิด — จริงๆ ถอนซี่ 15)
- ผู้ป่วยมาคลินิก B ต้องการอุดฟันซี่ 16
- ระบบขึ้น Error ว่า "ไม่ตรงเงื่อนไข" เพราะซี่ 16 ถูกบันทึกว่าถอนไปแล้ว

## กลไกระบบ

ระบบตรวจสอบก่อนจ่าย (Pre-Payment Audit / OSR) ทำงานที่ **15:00 น. ของทุกวัน** หลังจากส่งข้อมูล:
- ถ้าบันทึกหัตถการในซี่ที่ระบบรู้ว่า "ถอนแล้ว" → ระบบ flag ทันที → ปฏิเสธ (D001)
- หน่วยบริการจะเห็นผลใน **หน้า OSR Review 15:00 น.**

## วิธีแก้ไข

เนื่องจากไม่สามารถบันทึกข้อมูลผ่านระบบปกติได้ หน่วยบริการต้องทำ **อุทธรณ์**:

1. เข้าระบบ **Appeal ของ OSR** (appel ของ osr)
2. ระบุในอุทธรณ์ว่าได้ให้บริการซี่ดังกล่าวแล้ว
3. แจ้งว่าข้อมูลที่ block เกิดจากการบันทึกผิดของคลินิกอื่น
4. สปสช. จะพิจารณา **waive เป็นรายกรณี**

> กรณีประเภทนี้เกิดขึ้นไม่บ่อย แต่สามารถเกิดได้ในเชิงปฏิบัติ

## ข้อควรระวัง

- **ต้องไม่แก้ไขข้อมูลในคลินิกอื่น** โดยไม่ได้รับอนุมัติ
- ระบบโดยหลักการ: หัตถการในซี่ไหนต้องบันทึกซี่นั้น
- ถ้ามีประวัติถอนฟันในระบบ → ระบบจะ flag หัตถการอื่นในซี่เดียวกันทุกครั้ง
