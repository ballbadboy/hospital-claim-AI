# การจัดการ User ใน One Service (ข้ามหน่วยงาน)

## Definition

ปัญหาการใช้ User รหัสผู้ใช้งานข้ามหน่วยงานในระบบของ สปสช. เกิดขึ้นเมื่อบุคลากรสาธารณสุข (เช่น พยาบาล แพทย์แผนไทย) ยังรับราชการประจำในโรงพยาบาล **และ** เปิดคลินิกนวัตกรรมพาร์ทไทม์ควบคู่กัน — ต้องแยก User ออกเป็น 2 ชุดตามหน่วยงาน

## ปัญหาที่พบบ่อย

### กรณีตัวอย่าง (โรงพยาบาลบัวลาย เขต 9)

- พยาบาลใช้รหัส Data Center เดิม (ขึ้นต้นด้วยปี 58) ในการลงทะเบียนคลินิกนวัตกรรม
- ต่อมาเมื่อระบบ One Service ปรับปรุงให้ต้องมี User เฉพาะผู้ประกอบการ → เกิด User ซ้ำซ้อน
- ผลที่ตามมา: เข้าระบบ NPRP (National Perinatal Registry Portal System) แล้วเกิด error **E002 "ICR_date value is NULL"** เนื่องจาก User ไม่มีบทบาทในระบบนั้น

## หลักการ: 1 คน = หลาย User (แยกตามหน่วย)

| หน่วยงาน | ประเภทการทำงาน | User ที่ใช้ | บทบาทในระบบ |
|----------|---------------|-----------|-------------|
| โรงพยาบาลประจำ (Full-time) | ราชการ | User Data Center เดิม | NPRP, HIS, Data Center |
| คลินิกนวัตกรรม (Part-time) | เอกชน | User One Service ใหม่ | M Care Plus, Seamless for DMS, CPP |

## วิธีแก้ไข (2 ทางเลือก)

### ทางที่ 1 — คงรหัสเดิมของคลินิก
1. ใช้รหัส Data Center เดิม (ปี 58...) เป็น User คลินิก
2. แจ้ง สปสช.เขต ให้ **สลับบทบาท** — เพิ่มบทบาท Data Center ให้ User ใหม่, ดึงบทบาทนวัตกรรมออกจากรหัสเดิม
3. เพิ่มบทบาท NPRP ให้ User ที่ใช้ทำงานที่โรงพยาบาล

### ทางที่ 2 — ขอ User ใหม่สำหรับคลินิก
1. ขอ User ใหม่จากระบบ One Service สำหรับคลินิก
2. กำหนดบทบาทเฉพาะ: M Care Plus, Seamless for DMS เท่านั้น
3. User เดิม (ราชการ) คงบทบาทเดิมรวมถึง NPRP

## การบันทึกบุคลากรใน CPP

**สำคัญ:** ต้องบันทึกสถานะบุคลากรใน CPP ให้ถูกต้อง

| หน่วยที่สังกัด | ประเภทบุคลากรที่บันทึก |
|---------------|----------------------|
| คลินิกนวัตกรรม (คลินิกพาร์ทไทม์) | **Part-time** พร้อมระบุชั่วโมง/สัปดาห์ตามใบอนุญาต สสจ. |
| โรงพยาบาลประจำ (ราชการ) | **Full-time** ปกติ |

ถ้าบันทึกคลินิกเป็น Full-time → โรงพยาบาลประจำไม่สามารถบันทึกบุคลากรในระบบประเมินประจำปีได้

## กรณีสังกัดหลายหน่วย (มากกว่า 2 แห่ง)

- **หน่วยหลัก (ทำงานมากสุด/เจ้าของ):** บันทึก Full-time + ขอ User ในฐานะผู้ประกอบการ
- **หน่วยอื่นๆ (Part-time):** ขอ User แยกต่างหาก, บันทึก Part-time + คำนวณชั่วโมง/สัปดาห์ให้ไม่ซ้ำซ้อน
- หลีกเลี่ยงบริการที่เกิดพร้อมกันในสองหน่วย → ระบบตรวจพบความผิดปกติ
