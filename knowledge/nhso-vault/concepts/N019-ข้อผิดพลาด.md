# Error N019 / IN019 — ข้อผิดพลาดในระบบหน่วยนวัตกรรม

## Definition

**IN019** (หรือ N019) คือ Error Code ที่ระบบของ สปสช. แจ้งเมื่อข้อมูลการปิดสิทธิ์ของหน่วยนวัตกรรมมีความไม่สอดคล้องระหว่าง **ประเภทบริการ** ที่เลือกกับ **Payment Type Code** ที่ระบุ หรือเมื่อระบบดึงรายการผิดในกรณีที่มีการปิดสิทธิ์หลายรอบพร้อมกัน

**ย่อมาจาก:** Innovation 019 → ตัว "IN" ย่อมาจาก Innovation (หน่วยนวัตกรรม)

## Payment Type Codes ที่เกี่ยวข้อง

| Code | ความหมาย | มีโควต้า? |
|------|-----------|-----------|
| `PG019` | OP (Outpatient) ของหน่วยนวัตกรรม | ใช่ |
| `PG006` / `PG001` | PP (ส่งเสริมป้องกัน) | ไม่มี |

## สาเหตุหลักของ IN019

### กรณีที่ 1: เลือก Payment Type ผิด
- บริการ C (ล้างแผล / กายภาพบำบัด / ทันตกรรม) ที่ควรเป็น `PG019` แต่เลือกผิดเป็น `PG006`
- หรือกลับกัน — บริการ PP ที่ควรเป็น `PG006` แต่เลือกเป็น `PG019`

### กรณีที่ 2: ปิดสิทธิ์หลายรอบในครั้งเดียวกัน (สาเหตุหลักของปัญหาในช่วง มี.ค. 2569)
- หน่วยบริการส่งปิดสิทธิ์ 2+ รอบพร้อมกัน เช่น ทำ Lab (B64) ครั้งแรก และ PP (PG006) ครั้งที่สอง
- ระบบดึงเฉพาะ **รายการสุดท้าย** มาประมวลผล
- ถ้ารายการสุดท้ายเป็น `PG019` → รายการ PP อาจถูกปฏิเสธด้วย IN19
- ถ้ารายการสุดท้ายเป็น `PG006` (PP) → รายการ OP บางส่วนอาจไม่ได้รับ

### กรณีที่ 3: ไม่มี FVS Log
- ไม่มีการสแกนใบหน้า (Face Verification) เก็บไว้ในระบบ
- ระบบตรวจสอบไม่ได้ว่ามีการยืนยันตัวตนหรือยังก่อนจ่าย

## How It Works — กระบวนการแก้ไข

### สำหรับกรณีที่ติด IN19 อยู่แล้ว (Retroactive Fix)
1. สปสช. จะ **re-run ข้อมูลทุกวันจันทร์** สำหรับรายการที่ติด IN19 ในสัปดาห์ก่อน
2. ระบบตรวจสอบว่ามี **FVS log (สแกนใบหน้า) อย่างน้อย 2 ครั้ง** หรือไม่
3. ถ้ามี log → ดึงมาจ่ายโดยอัตโนมัติ **ไม่ต้องยื่นอุทธรณ์**
4. ถ้าไม่มี log → ยังไม่จ่าย (ต้องตรวจสอบเพิ่มเติม)

### สำหรับการป้องกัน (Forward)
1. สแกนใบหน้า (FVS) ก่อนทุกครั้ง **อย่างน้อย 2 ครั้ง** เพื่อให้ระบบเก็บ log
2. ถ้าสแกนไม่ผ่าน → ใช้วิธียืนยันตัวตนแทน ([[concepts/กลุ่มเปราะบาง-authen]])
3. เลือก Payment Type ให้ถูกต้องตามประเภทบริการ
