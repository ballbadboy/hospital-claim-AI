# C936 — ข้อมูลสิทธิ์ขัดกัน (ย้ายหน่วยเกิดสิทธิ์ทันที)

## Definition

C936 คือรหัสปฏิเสธที่เกิดขึ้นเมื่อหน่วยบริการเลือกเงื่อนไขการเรียกเก็บ "ย้ายหน่วยบริการแล้วเกิดสิทธิ์ทันที" แต่ข้อมูลสิทธิ์ที่ระบบหลังบ้านตรวจสอบได้ขัดแย้งกับสิทธิ์ที่ปรากฏในหน้าจอ eclaim ขณะบันทึก

## สาเหตุหลัก

### ระบบ SRM ล่ม (26–28 กันยายน 2568)
ในช่วงวันที่ 26–28 กันยายน 2568 สปสช. ปรับระบบทะเบียนไปสู่ระบบ SRM ซึ่งส่งผลให้:
- ระบบตรวจสอบสิทธิ์และปิดสิทธิ์มีปัญหาชั่วคราว
- ผู้รับบริการที่มีสิทธิ์บัตรทองอาจปรากฏเป็น "สิทธิ์ว่าง" ในระบบ
- หน่วยบริการจึงเลือกเงื่อนไข "ย้ายหน่วยเกิดสิทธิ์ทันที" แทนเงื่อนไขปกติ

### กลไกที่ทำให้เกิด C936
เมื่อส่งข้อมูลผ่าน **Financial Data Hub** (ไม่ใช่หน้าจอ eclaim โดยตรง):
1. ระบบไม่ได้ตรวจสอบสิทธิ์ ณ ขณะบันทึก
2. ข้อมูลมาตรวจสอบสิทธิ์ "หลังบ้าน" อีกครั้ง
3. สิทธิ์ที่พบหลังบ้านขัดแย้งกับเงื่อนไขที่เลือก → ติด C936

## วิธีแก้ไข

1. **ตรวจสอบสิทธิ์ ณ วันที่รับบริการ** — เข้าระบบตรวจสอบสิทธิ์ปัจจุบัน ดูว่า ณ วันนั้นผู้รับบริการมีสิทธิ์ที่ไหน
2. **กรณีสิทธิ์อยู่ที่หน่วยเราแล้ว:** ไม่ต้องเลือกเงื่อนไขย้ายสิทธิ์ทันที — บริการนั้นอยู่ในเครือข่ายปกติ ไม่สามารถเบิกแยกสปสช.
3. **กรณีสิทธิ์อยู่ที่หน่วยอื่น:** สามารถเลือกเงื่อนไขย้ายหน่วยได้ แต่ต้องส่งข้อมูลใหม่ผ่าน Financial Data Hub อีกครั้ง (ชุดข้อมูลเดิม)
4. **ข้อควรระวัง:** ถ้าเบิก Walk-in ในจังหวัดเดียวกันหรือ "เหตุสมควร" ระบบจะจ่าย 0 บาท

## เงื่อนไขการย้ายสิทธิ์
ผู้มีสิทธิ์สามารถย้ายหน่วยบริการแล้วเกิดสิทธิ์ทันทีได้ **ปีละ 4 ครั้ง**

## กรณีทั่วไป (ไม่ใช่ช่วงระบบล่ม)

จากไลฟ์ครั้งที่ 81 — C936 เกิดได้แม้ระบบปกติ หากหน่วยบริการไม่ทำธุรกรรมย้ายสิทธิ์ให้เสร็จก่อนให้บริการ:

1. **สถานะสิทธิ์ก่อนย้าย = 0.4** (สิทธิ์อยู่ที่หน่วยอื่น)
2. ทำการย้ายสิทธิ์ใน **SRM** → สถานะเปลี่ยนเป็น **011** (ย้ายสำเร็จ)
3. เฉพาะหลัง status 011 เท่านั้น → จึงทำการให้บริการและปิดสิทธิ์ → ส่งข้อมูลเบิก
4. ถ้า OPD case เปิดก่อนที่ status จะเปลี่ยน → ติด C936 ทันที

**วันที่ + เวลาย้ายสิทธิ์สำเร็จ** ต้องมาก่อนวันที่+เวลาเปิด OPD case เสมอ
