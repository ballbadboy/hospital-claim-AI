# Dashboard CD9 — ระบบสารสนเทศข้อมูลปฏิเสธการจ่าย

## Definition

Dashboard CD9 คือ **ระบบ Dashboard สารสนเทศข้อมูลปฏิเสธการจ่าย** ที่ สปสช. พัฒนาขึ้นเพื่อให้ผู้บริหารและหน่วยบริการสามารถตรวจสอบรายการที่ถูกปฏิเสธ (C-Code และ 9-Code) ได้ทั้งในระดับเขตสุขภาพและระดับโรงพยาบาล

## ข้อมูลที่แสดง

- ปริมาณรายการปฏิเสธแยกตาม **เขตสุขภาพ** (13 เขต)
- แยกตาม **รหัส C** และ **รหัส 9**
- แสดงกราฟรายเดือน (พีคช่วงกุมภาพันธ์ และกันยายน)
- แสดงตามประเภทหน่วยบริการ (ส่วนใหญ่รัฐสังกัด สป.)
- แสดงตามสิทธิ์ (ส่วนใหญ่สิทธิ์ UC และสวัสดิการข้าราชการ)
- **ค่าใช้จ่ายเรียกเก็บ** (ถ้าแก้ไขแล้วจะได้รับเงินคืน)

## วิธีใช้งาน

1. เลือกเขตสุขภาพที่ต้องการ → คลิกเข้าไป
2. ดูภาพรวมรายการปฏิเสธในเขต (เช่น เขต 1 ติด C30 เกือบ 150,000 รายการ)
3. ระบุ **รหัสหน่วยบริการ** เพื่อดูเฉพาะโรงพยาบาลนั้น
4. **เรียงตามค่าใช้จ่ายเรียกเก็บ** (มากไปน้อย) เพื่อจัดลำดับสิ่งที่ต้องแก้ไขก่อน
5. อ่านคำอธิบายวิธีแก้ไขในแต่ละรหัส C

## ตัวอย่างข้อมูล เขต 1

| รหัส | จำนวนรายการ | หมายเหตุ |
|------|-------------|----------|
| C30  | ~150,000 | มากที่สุด — แก้ผ่าน Appeal 349 |
| C (อื่นๆ) | ~49,000 | รวมทุก C-Code |
| 9-Code | ~3,500 | |
| **รวมทั้งหมด** | **400,000+** | ยังไม่เข้าสู่การประมวลผล |

## การจัดลำดับความสำคัญ

สปสช. แนะนำให้หน่วยบริการจัดลำดับ:
1. **รายการที่มีค่าใช้จ่ายเรียกเก็บสูงสุด** → แก้ไขก่อน
2. กรณีติดอุปกรณ์/อวัยวะเทียม → ถ้าไม่แก้จะยุติทั้งเคส (รวมบริการพื้นฐาน)
3. C30 จำนวนมาก → ใช้ Appeal 349 (มกราคม 2569)
4. C998 ข้ามปี → ทำหนังสือถึงเลขาธิการ
