# กระบวนการอุทธรณ์ OSR (On-Screen Review Appeal)

## Definition

**OSR (On-Screen Review)** คือกระบวนการที่ Auditor ของเขตสุขภาพทำการตรวจสอบข้อมูลเบิกจ่ายก่อนการจ่ายชดเชย เมื่อระบบตรวจพบข้อมูลที่ต้องสงสัย ข้อมูลชุดนั้นจะถูกส่งให้ Auditor พิจารณาแทนที่จะออก IEP โดยตรง

## Flow กระบวนการ

```
หน่วยบริการส่งข้อมูลเบิก
    ↓
ประมวลผลทุกเที่ยงคืน
    ↓
06:00 น. วันถัดไป — ข้อมูลผ่าน → รอออก IEP
                 — ข้อมูลต้องสงสัย → Auditor พิจารณา 06:00–15:00 น.
    ↓
บ่ายวันถัดไป — Auditor แจ้งผล
    ├── ผ่าน → เข้ากระบวนการจ่ายปกติ
    └── ปฏิเสธ D001 → แจ้งหน่วยบริการทราบ
```

## ระยะเวลาอุทธรณ์

- หน่วยบริการมีเวลา **10 วัน** นับจากวันที่ระบบแจ้งการปฏิเสธ D001
- เข้าอุทธรณ์ผ่านเว็บแอปพลิเคชัน **Appeal OSR** (ap-osr.nhso.go.th หรือเมนูใน eClaim)

## 3 ทางเลือกเมื่อถูกปฏิเสธ

| ทางเลือก | รายละเอียด | ผลลัพธ์ |
|---------|-----------|--------|
| 1. แก้ไขข้อมูลตามคำแนะนำ | เลือกเหตุผล → แก้ข้อมูลต้นทาง → ส่งใหม่ | ข้อมูลเข้าสู่ระบบ On-Screen อีกรอบ |
| 2. ขอแนบเวชระเบียนเพิ่มเติม | อัปโหลดเอกสารประกอบ | Auditor พิจารณาด้วยข้อมูลเพิ่มเติม |
| 3. ไม่ขออุทธรณ์ | ยอมรับการปฏิเสธ | ข้อมูลชุดนั้นไม่ได้รับการชดเชย |

## กระบวนการหลังแก้ไขข้อมูล (ทางเลือกที่ 1)

1. สถานะ D001 ถูกส่งกลับไปยัง Financial Data Hub ว่า "ถูกปฏิเสธโดย Auditor"
2. หน่วยบริการแก้ไขข้อมูลต้นทางในระบบ eClaim หรือ HIS
3. ส่งข้อมูลเข้ามาใหม่ผ่านระบบเคลมตามปกติ
4. ข้อมูลเข้าสู่รอบ On-Screen Review อีกครั้ง
5. ถ้าผ่าน → รอออก Statement ตามรอบ 15 วัน

## ตัวอย่างการคำนวณเวลา

> หน่วยบริการทราบผลการปฏิเสธวันศุกร์ 15:00 น.
> → แก้ไขข้อมูลในวันเสาร์–อาทิตย์
> → ส่งข้อมูลใหม่วันจันทร์
> → ระบบประมวลผล → ออกรายงานผล IEP วันพุธ (ถ้าผ่าน)
> → เข้ารอบ Statement 15 วัน ตามปฏิทิน
