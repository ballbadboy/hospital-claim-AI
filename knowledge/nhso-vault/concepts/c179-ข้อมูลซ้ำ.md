# C179 — รายการข้อมูลซ้ำซ้อน

## Definition

รหัสปฏิเสธ C179 เกิดขึ้นเมื่อระบบ EClaim ตรวจพบว่าข้อมูลที่ส่งเข้ามาซ้ำซ้อนกับข้อมูลที่มีอยู่แล้วในระบบ หรือซ้ำกับรายการที่พร้อมจ่ายเงินแล้ว

## How It Works

### สาเหตุที่เกิด C179
1. บันทึกข้อมูลซ้ำกับรายการที่ได้รับการจ่ายชดเชยแล้ว
2. บันทึกซ้ำกับรายการที่อยู่ระหว่างรอจ่าย
3. ผู้ป่วยในถูก Admit วันเดียวกันมากกว่า 1 ครั้งในหน่วยบริการเดียวกัน

### วิธีแก้ไข
**กรณีที่ 1 — ซ้ำกับรายการที่จ่ายแล้ว:**
- ไม่จำเป็นต้องแก้ไข รายการซ้ำจะค้างอยู่ในระบบโดยไม่มีผลเสีย เพราะรายการอื่นที่เป็นรายการเดิมได้รับการจ่ายแล้ว

**กรณีที่ 2 — ต้องการแก้ไขให้สะอาด:**
1. ในเรคคอร์ดที่ติด C179 ให้ขยับวันที่รับบริการเป็นวันอื่น (เพื่อไม่ให้ซ้ำซ้อน)
2. เลือกเงื่อนไข UCC = 2 (ไม่ประสงค์เบิก)
3. ส่งผ่าน Financial Data Hub อีกครั้ง
4. ระบบจะปรับสถานะรายการนั้นเป็น "ไม่ประสงค์เบิก"

### สิ่งที่ควรรู้
- รหัส C ใน EClaim **ไม่หาย** เนื่องจากเป็น Digital Fingerprint หลักฐานการส่งข้อมูล
- ให้ยึดสถานะใน IEP (Individual Electronic Payment) ของ EClaim เป็นหลัก ไม่ใช่สถานะใน FDS
- ครั้งถัดไปจะมีการนำเรื่อง C179 มาอธิบายละเอียดพร้อมหน้าจอสาธิต
