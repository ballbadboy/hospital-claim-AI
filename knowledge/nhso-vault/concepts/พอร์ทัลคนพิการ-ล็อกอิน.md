# Disability Portal — การ Login ด้วย Thai ID (Multifactor)

## Definition

**Disability Portal** คือระบบบันทึกข้อมูลบริการฟื้นฟูสมรรถภาพของคลินิกกายภาพบำบัดและหน่วยบริการที่ให้บริการ IMC (Intermediate Care) ในระบบ สปสช. ตั้งแต่ต้นปี 2569 ระบบนี้ถูกปรับให้ login ด้วย **Thai ID (ไทยดี / D.DOPA) แบบ Multifactor** แทน username+password เดิม

## ปัญหาที่พบบ่อย — กรณีมีหลาย User

### สาเหตุ

เมื่อ login ด้วย Thai ID (เลขบัตรประชาชน) ระบบจะยึด **PID เป็น key** แทน username ดังนั้นถ้าบุคลากรคนเดียวมีหลาย user (เช่น ผูกกับโรงพยาบาลและคลินิกกายภาพบำบัดคนละ user) ระบบจะวิ่งไปหา **user ตัวล่าสุดที่ active** อัตโนมัติ

ถ้า user ที่ระบบเลือกให้ไม่มีบทบาท (role) สำหรับ Disability Portal ก็จะเข้าไม่ได้

### วิธีแก้ไข (2 วิธี)

**วิธีที่ 1:** เพิ่มบทบาท (role) ให้ครบทุก user ที่มีอยู่ เพื่อให้ไม่ว่าระบบจะเลือก user ไหน ก็เข้าได้ทุก system

**วิธีที่ 2:** สลับโปรไฟล์ก่อน login
1. เข้า **portal.nhso.go.th** (ระบบ SRM Portal)
2. Login ด้วย username+password ก่อน
3. กดเมนูด้านบน → เลือก "สลับโปรไฟล์"
4. เลือก user ที่ผูกกับคลินิกกายภาพบำบัดที่ต้องการ
5. จากนั้นค่อยเข้า Disability Portal ด้วย Thai ID — ระบบจะจำว่า PID นี้ผูกกับ user ที่เลือกไว้

## ระบบที่ใช้ Thai ID Multifactor แล้ว (ณ มีนาคม 2569)

| ระบบ | สถานะ |
|------|-------|
| New Authen (ยืนยันตัวตน) | ใช้แล้ว |
| AMED (คลินิกพยาบาล/แผนไทย) | ใช้แล้ว |
| Disability Portal | ใช้แล้ว |
| ระบบอื่นๆ | ทยอยปรับในปี 2569 |

## ปัญหา Session ขาว (White Screen) หลัง Thai ID Scan

เกิดเมื่อ:
- Login หลายเครื่องหรือหลาย tab พร้อมกัน
- Session ค้างจากการใช้งานก่อนหน้า

วิธีแก้: ปิดทุก tab/browser ทั้งหมด → รีสตาร์ทเครื่อง → Login ใหม่ที่เครื่องเดียว/tab เดียว

## สำคัญ

Thai ID scan ออกแบบให้ใช้ได้แค่ **1 device** ต่อ session เนื่องจากผูกกับ PID — ถ้าแชร์ session ให้คนอื่นจะถูก logout อัตโนมัติ
