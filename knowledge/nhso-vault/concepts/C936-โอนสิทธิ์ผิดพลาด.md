# C936 — ข้อมูลเบิกจ่ายไม่ตรงกับระบบลงทะเบียนย้ายสิทธิ์

## Definition

**C936** คือ C-code ที่เกิดขึ้นเมื่อหน่วยบริการส่งข้อมูลเบิกจ่ายมาในกรณี **ผู้ป่วยย้ายหน่วยบริการและเกิดสิทธิ์ทันที** แต่ไม่ได้ทำการ **ย้ายสิทธิ์ในระบบ SRM** ให้เสร็จก่อนส่งข้อมูลเบิก ทำให้ระบบ e-Claim ตรวจสอบกับระบบลงทะเบียนแล้วไม่พบสิทธิ์ที่ถูกต้อง

## สาเหตุ

กรณีผู้ป่วยย้ายหน่วยบริการ + เกิดสิทธิ์ทันที:

1. ห้องลงทะเบียน/ห้องบัตร ยังไม่ทำการย้ายสิทธิ์ใน **ระบบ SRM (ระบบใหม่)**
2. หน่วยบริการส่งข้อมูลเบิกจ่ายผ่านระบบ e-Claim → Financial Data Hub
3. ระบบ e-Claim นำข้อมูล (วันเริ่มบริการ, วันสิ้นสุดบริการ) ไปตรวจสอบกับระบบลงทะเบียน
4. ไม่พบการย้ายสิทธิ์ → **ติด C936**

## ผลที่ตามมาถ้าส่งข้อมูลเดิมซ้ำ

ถ้าหน่วยบริการส่งข้อมูลชุดเดิมเข้ามาอีกครั้งโดยไม่แก้ไขต้นเหตุ → ติด **C179** (ซ้ำซ้อน)

## วิธีแก้ไข

1. **ตรวจสอบก่อน** ว่าห้องลงทะเบียน/ห้องบัตรได้ทำการย้ายสิทธิ์ในระบบ SRM ในวันนั้นเรียบร้อยแล้วหรือไม่
2. ถ้ายังไม่ได้ย้ายสิทธิ์ → **ดำเนินการย้ายสิทธิ์ใน SRM ก่อน**
3. หลังย้ายสิทธิ์เรียบร้อยแล้วจึงส่งข้อมูลเบิกจ่ายใหม่

> ลำดับที่ถูกต้อง: **ย้ายสิทธิ์ SRM ก่อน → จึงส่งข้อมูล e-Claim**

## ป้องกัน

- กำหนด Flow ภายในโรงพยาบาล: ห้องบัตรต้องยืนยัน SRM เสร็จก่อนส่งเรื่องให้แผนกบัญชี/การเงินส่งข้อมูล
- ตรวจสอบสถานะสิทธิ์ใน SRM ของผู้ป่วยก่อนส่ง e-Claim ทุกครั้ง
