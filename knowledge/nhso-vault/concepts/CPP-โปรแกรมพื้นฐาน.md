# CPP — โปรแกรมข้อมูลพื้นฐานและศักยภาพหน่วยบริการ

## Definition

**CPP (Clinic Profile Program / โปรแกรมข้อมูลพื้นฐานและศักยภาพ)** คือระบบที่ สปสช. ใช้บันทึกข้อมูลพื้นฐานของหน่วยบริการนวัตกรรม ได้แก่ ข้อมูลทั่วไปของสถานพยาบาล, บุคลากร, และวันเวลาให้บริการ ระบบนี้เชื่อมโยงกับ OSS และสภาวิชาชีพ

## CPP Version ใหม่ (2569)

### การเปลี่ยนแปลงสำคัญ

| | เวอร์ชันเก่า | เวอร์ชันใหม่ |
|--|------------|------------|
| วันหมดอายุ | สิ้นสุด 31 ธ.ค. 2568 | เริ่มใช้ 30 ธ.ค. 2568 |
| บุคลากรพยาบาล | ไม่รองรับรายบุคคล | รองรับรายบุคคล (4.1 / 4.2.26) |
| บุคลากรกายภาพ | ไม่รองรับรายบุคคล | รองรับรายบุคคล |
| การเชื่อมสภาวิชาชีพ | ไม่มี API | มี API validation ทันที |

### ผลกระทบ

- **เวอร์ชันเก่าใช้ไม่ได้อีกต่อไป** นับจาก 31 ธ.ค. 2568
- หน่วยที่เคยบันทึกใน CPP เวอร์ชันเก่า → ต้องบันทึกใหม่ทั้งหมดในเวอร์ชันใหม่
- CPP เวอร์ชันใหม่เป็นส่วนหนึ่งของกระบวนการ **Reactivate** ใน OSS

## ข้อมูลที่ต้องบันทึกใน CPP

### 1. ข้อมูลทั่วไปสถานพยาบาล
- ชื่อคลินิก, ที่อยู่, ใบอนุญาต (สพ.6 / สพ.7)
- พิกัด GPS (Latitude/Longitude) — **ต้องทำที่คลินิก**

### 2. บุคลากร
- เลขบัตรประชาชน + เลข ว. (ใบประกอบวิชาชีพ)
- ประเภทการทำงาน: Full-time / Part-time
- วัน-เวลาปฏิบัติงาน (ใช้คำนวณ Man-hour และ Workload)
- ข้อมูลต้องตรงกับ **สพ.6**

### 3. การเชื่อมสภาวิชาชีพ (API)

เมื่อบันทึกเลข ว. ระบบ OSS จะ call API ไปยังสภาวิชาชีพโดยอัตโนมัติ:
- สำเร็จ → ปุ่มสีเขียว = Reactivate เรียบร้อย
- ไม่สำเร็จ → ปุ่มสีแดง = ต้องติดต่อสภาวิชาชีพ

## สิทธิ์ในการเข้าถึงระบบ CPP

เฉพาะผู้ที่มีสิทธิ์ทำ Reactivate เท่านั้น:
- **ผู้ประกอบกิจการ** (เจ้าของคลินิก/กรรมการที่ลงทะเบียนกับกรมพัฒนาธุรกิจการค้า)
- **ผู้รับมอบอำนาจ** ที่ได้รับอนุมัติจาก สปสช. แล้ว

> ข้อควรระวัง: Admin หน่วยบริการ (user ปกติ) ไม่มีสิทธิ์เข้า CPP เพื่อ Reactivate — ต้องเป็น user ที่ผูกกับผู้มีอำนาจที่ยืนยันกับกรมพัฒนาธุรกิจการค้าแล้วเท่านั้น

## เหตุผลที่ต้องบันทึกบุคลากรรายบุคคล

ระบบใหม่เชื่อมกับ **Heink** เพื่อให้บุคลากรเข้าดูประวัติการรักษาของผู้ป่วย ซึ่งเป็น sensitive data ตาม PDPA — สปสช.จึงต้องรู้ว่าใครเป็นผู้เข้าถึงข้อมูล
