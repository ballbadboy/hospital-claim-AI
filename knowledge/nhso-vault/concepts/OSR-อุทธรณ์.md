# OSR Appeal — ระบบตรวจสอบและอุทธรณ์ก่อนจ่าย (On Screen Review)

## Definition

**OSR (On Screen Review)** คือระบบที่ สปสช. ใช้ **ตรวจสอบข้อมูลต้องสงสัย** ก่อนการจ่ายชดเชย โดย Auditor (K) จะตรวจสอบข้อมูลบริการที่หน่วยบริการส่งเบิกเข้ามา หากต้องการข้อมูลเพิ่มเติม ระบบจะแจ้งเตือนหน่วยบริการผ่าน LINE Alert

**OSR Appeal** คือฟังก์ชันที่หน่วยบริการใช้ **ส่งหลักฐานตอบ** คำขอของ Auditor เช่น เวชระเบียน, ระยะเวลาเปิดให้บริการ ฯลฯ

## กระบวนการทำงาน

```
หน่วยบริการส่งเบิก
→ Auditor ตรวจข้อมูลต้องสงสัย
→ ระบบ OSR ส่ง LINE Alert แจ้งหน่วยบริการ
→ หน่วยบริการเข้า OSR → แนบหลักฐานภายใน 10 วัน
→ ข้อมูลถูกนำไปประมวลผลรอบถัดไป
```

## ระยะเวลาสำคัญ

- **10 วัน** นับจากได้รับ LINE Alert — หน่วยบริการต้องส่งหลักฐานภายในกำหนด
- ถ้าเกิน 10 วัน → ข้อมูลอาจ **หลุดรอบ** การประมวลผล → ล่าช้าออกไปอีกรอบ

## คำแนะนำจาก สปสช.

**ให้ตรวจ OSR ทุกวัน** ก่อนดูรายงาน Seamless for DMS โดย:
1. ตั้งค่า LINE Alert ใน OSR ให้แจ้งเตือนทุกวัน
2. เมื่อได้รับแจ้งเตือน → เข้า OSR Appeal ทันที
3. อัปโหลดหลักฐานที่ระบบขอ
4. ถ้าทำได้เร็ว → ข้อมูลจะประมวลผลในรอบถัดๆ ไปเลย

> "ก่อนที่จะไปดูรายงานใน Seamless for DMS ให้ LINE Alert เข้าไปดูใน OSR Appeal ก่อนทุกวัน" — คุณมานพ โยเฮือง (ครั้งที่ 24)

## ความสัมพันธ์กับการจ่ายเงิน

OSR อยู่ในขั้นตอนก่อน Seamless for DMS:

```
OSR Appeal (ตรวจสอบ/ส่งหลักฐาน)
→ Seamless for DMS / Simulate (ออกรายงาน IEP/Statement)
→ Smart Money Transfer (ยืนยันยอดโอน)
```

## การตั้ง LINE Alert ใน OSR

ระบบ OSR มีฟังก์ชันให้ตั้งค่า LINE Alert เพื่อ:
- แจ้งเตือนเมื่อมีข้อมูลบริการถูกเลือกตรวจสอบ
- แจ้งเตือนเมื่อมีข้อมูลต้องการหลักฐานเพิ่มเติม
