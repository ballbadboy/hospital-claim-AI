# การปิดสิทธิ์ (Rights Closure / Check-out)

## Definition

การปิดสิทธิ์ คือกระบวนการ**ยืนยันว่าประชาชนได้รับบริการจริง** ณ หน่วยบริการ เป็นขั้นตอนที่หน่วยบริการต้องดำเนินการหลังให้บริการเสร็จสิ้น เพื่อให้สปสช. สามารถเบิกจ่ายค่าชดเชยได้

## หลักการ

- การปิดสิทธิ์ = check-and-balance ว่าคนไข้มาจริงและได้รับบริการจริง
- สปสช. ยืนยันว่าเป็นกลไกจำเป็น แม้หน่วยบริการจะสะท้อนว่าเป็นภาระงาน

## 7 วิธีปิดสิทธิ์

สปสช. กำหนด 7–8 วิธีสำหรับหน่วยบริการ วิธีหลักที่แนะนำ:

| วิธีที่ | ชื่อ | เหมาะกับ |
|---------|------|----------|
| 1–6 | วิธีปกติผ่านระบบต่างๆ | ทั่วไป |
| **7** | **Minimal Data Set** | หน่วยบริการสังกัด สป. |

### วิธีที่ 7 — Minimal Data Set

- บันทึกผ่าน 6 fields ที่กำหนด
- **Field สำคัญ: วันที่และเวลา** — ต้องเป็นวันที่/เวลาสิ้นสุดบริการจริง

## ปัญหาที่พบบ่อย: เวลาปิดสิทธิ์ไม่สอดคล้อง (C305)

**สาเหตุ:** ระบบ HIS ของโรงพยาบาลกวาดข้อมูลอัตโนมัติช่วง 21:00–23:00 น. ทำให้เวลาปิดสิทธิ์เร็วกว่าเวลาสิ้นสุดบริการจริง

**ตัวอย่าง:**
- คนไข้มา 08:00 น. เสร็จบริการ 14:00 น.
- ระบบกวาดข้อมูล 22:00 น. → เวลาปิดสิทธิ์ = 22:00 น. (ผิด)
- ควรใช้: **14:00 น.** (เวลาสิ้นสุดบริการจริง)

**วิธีแก้:**
- ตั้งค่าให้ระบบ HIS ดึงเวลาสิ้นสุดบริการจริงจาก EMR
- ถ้าติด C305 แล้ว → รอรอบอุทธรณ์จาก สปสช.

## กรณี OP Anywhere (Walk-in)

- ผู้ป่วยมารับบริการข้ามจังหวัด
- บันทึกเวลาปิดสิทธิ์เป็นเวลาสิ้นสุดบริการ **ทุกแผนก** ในวันนั้น
- ถ้าผู้ป่วยมีหลายแผนก → ใช้เวลาสุดท้ายที่แผนกสุดท้ายเสร็จ

## ข้อร้องเรียนจากหน่วยบริการ

หน่วยบริการสะท้อนว่าการปิดสิทธิ์:
- ยุ่งยากเกินความจำเป็น โดยเฉพาะเวรบ่าย-เวรดึก
- ภาวะฉุกเฉิน/คนไข้ไม่นำบัตรประชาชน → ยิ่งยาก
- ภาระงานเพิ่มมากบนบุคลากรที่มีอยู่

สปสช. รับทราบและเก็บข้อเสนอแนะไว้พิจารณา

## ปัญหา C305 — ข้อมูลเชิงปริมาณ (ครั้งที่ 86)

จากข้อมูลที่ สปสช. รวบรวม ณ วันที่ 30 มกราคม 2569:

| กลุ่ม | จำนวน records | การจัดการ |
|-------|--------------|-----------|
| ทั้งหมด | ~1,500,000 | — |
| เกิดจากระบบ สปสช. | **700,000** | สปสช. แก้ให้อัตโนมัติ |
| หน่วยบริการต้องอุทธรณ์ | **800,000** | ขึ้นเว็บไซต์รายหน่วย เปิดอุทธรณ์ 1–28 ก.พ. 2569 |

### ปัญหาปิดสิทธิ์ซ้ำผ่าน EDC

- ผู้ป่วยไปหลายแผนก → ได้ KM Code หลายเลข → บันทึกคั่นด้วยอักขระพิเศษ
- ระบบตรวจพบ KM Code ไม่ถูกต้อง → ติด C305
- **วิธีแก้ของ สปสช.:** ถ้า 15% ของ C305 ทั้งหมดของหน่วยเป็น pattern นี้ → คัดเฉพาะ KM Code ครั้งสุดท้ายมาประมวลผล

### ปัญหารอยต่อระหว่างวัน (Midnight Boundary)

- ผู้ป่วยรับบริการ 23:00 เสร็จหลังเที่ยงคืน → ระบบตัดรอบก่อนเที่ยงคืน → ติด C305
- **วิธีแก้:** ส่งข้อมูลภายในวันเดียวกัน ระบบจะขยายเวลาตรวจสอบเป็น 24 ชั่วโมง

### รหัส IN22

- ความหมาย: ปิดสิทธิ์ไม่ผ่าน FVS หรือ Line OA
- เกิดกับบริการ PP ที่ปิดผ่าน Krungthai Digital Health Platform (ยังไม่รองรับหน่วยนวัตกรรม)
- สปสช. จะ waive และ reprocess ให้

### รหัส C30 (eClaim)

- สาเหตุหลัก: วัน/เวลาปิดสิทธิ์ไม่ตรงกับเวลาสิ้นสุดบริการจริง
- ต้องตรวจสอบว่าเวลาปิดสิทธิ์ = เวลาสิ้นสุดบริการ (ไม่ใช่เวลาที่ส่งข้อมูล)
- ข้อมูลปีงบ 2569 (ตั้งแต่ 1 ม.ค. 2569): ไม่มีติด C305 แล้ว ระบบแจ้ง error ทันทีตอนส่งข้อมูล

## ช่องทางอุทธรณ์ C305

- **เว็บ:** ap349.nhso.go.th
- **ระยะเวลา:** 1–28 กุมภาพันธ์ 2569
- **เงื่อนไข:** อัปโหลดเอกสารแยกรายบุคคล + ใส่เหตุผลรายแถว + มีลายมือผู้มีอำนาจลงนาม
- **ระยะเวลาตรวจสอบ:** 30 วัน (มาตรฐาน) หลังอนุมัติออก Statement ปกติ
- **กรณีฟอกไต AVF/AVG:** อุทธรณ์ผ่าน eClaim → ระบบ EPL (แยกต่างหาก)
