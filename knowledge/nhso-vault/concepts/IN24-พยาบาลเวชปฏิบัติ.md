# รหัสปฏิเสธ IN24 — ไม่มีพยาบาลเวชปฏิบัติ

## Definition

**IN24** = รหัสปฏิเสธบริการ CI (ตรวจรักษาโรคเบื้องต้น Common Illness) สำหรับหน่วยบริการรับส่งต่อด้านการพยาบาล เนื่องจากระบบไม่พบ**พยาบาลวิชาชีพสาขาเวชปฏิบัติทั่วไปและรักษาโรคเบื้องต้น (รหัส 4.2.26)** ที่ขึ้นทะเบียนไว้ในหน่วยบริการ

## เงื่อนไขการให้บริการ CI

บริการ CI (Common Illness) ในคลินิกพยาบาลและผดุงครรภ์กำหนดให้ต้องมี:
- **พยาบาลวิชาชีพประเภท 4.2.26** (พยาบาลเวชปฏิบัติทั่วไปและรักษาโรคเบื้องต้น) ที่บันทึกในระบบ OSS/One Service
- พยาบาลต้องเลือก**ประเภทที่ถูกต้อง**ในขั้นตอนการขึ้นทะเบียน

## สาเหตุหลักที่ติด IN24

1. **บันทึกประเภทพยาบาลไม่ถูกต้อง** — ไม่ได้เลือกประเภท 4.2.26 ตอน reactivate
2. **ข้อมูลใน One Service Register ไม่อัปเดต** — แม้จะเป็นพยาบาลเวชปฏิบัติจริง แต่ระบบยังไม่แสดง
3. **API สภาการพยาบาล:** สปสช.ตรวจสอบกับ API ของสภาการพยาบาลอัตโนมัติ ถ้าข้อมูลตรงก็จะนำมาประกอบการประมวลผล แต่บางกรณียังพบ IN24 อยู่

## แนวทางแก้ไข

### ระยะสั้น (ระหว่างรอแก้ข้อมูล)
- **งดให้บริการ C (CI)** เพื่อป้องกัน IN24 สะสม
- ให้บริการประเภทอื่นแทน: บริการพื้นฐาน (OPD) และบริการ PP (สร้างเสริมสุขภาพ)

### ระยะกลาง
- รอสปสช.เปิด **One Service Register** ให้หน่วยบริการเข้ามาแก้ไขข้อมูลอีกครั้ง (กำลังกำหนดวัน)
- ตรวจสอบใน CPP ว่าบันทึกบุคลากรเป็นประเภท 4.2.26 ถูกต้องหรือไม่

### ระยะยาว
- บริการ CI ที่ให้ไปแล้วและติด IN24 สปสช.จะนำข้อมูลมาตรวจสอบและประมวลผลจ่ายให้ต่อไป (แจ้งเป็นระยะ)

## วิธีตรวจสอบพยาบาลประเภท 4.2.26 ใน CPP

1. เข้าระบบ CPP ด้วย User คลินิก
2. เลือก "ข้อมูลบุคลากร" → กลุ่มข้อมูลศักยภาพ
3. ดูหมวด **4 (พยาบาล)** → หมวดย่อย **4.2** (พยาบาลวิชาชีพเฉพาะทาง) → **4.2.26**
4. ถ้าพบ 1 ท่านขึ้นไป → บันทึกสถานะ Full-time หรือ Part-time ให้ถูกต้อง

## ความสัมพันธ์กับ IN23

| รหัส | สาเหตุ | วิธีแก้ |
|------|--------|--------|
| IN23 | ยังไม่ reactivate เชื่อมฐานวิชาชีพ | ทำ reactivate ก่อน |
| **IN24** | reactivate แล้วแต่ประเภทพยาบาลไม่ถูก / ไม่พบ 4.2.26 | แก้ข้อมูลใน One Service Register |
