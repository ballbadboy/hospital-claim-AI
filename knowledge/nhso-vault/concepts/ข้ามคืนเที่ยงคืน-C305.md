# ปัญหารอยต่อระหว่างวัน (Midnight Boundary) — C305

## Definition

**ปัญหารอยต่อระหว่างวัน** คือกรณีที่ผู้ป่วยเริ่มรับบริการในวันหนึ่งแต่สิ้นสุดหลังเที่ยงคืน ทำให้ข้อมูลการปิดสิทธิ์ตกข้ามวัน และระบบของ สปสช. ซึ่งตัดรอบก่อนเที่ยงคืนตรวจไม่พบการปิดสิทธิ์ → ติด C305

## สถานการณ์ที่เกิด

```
ผู้ป่วยรับบริการ 23:00 น.
บริการสิ้นสุด 00:30 น. (วันถัดไป)
ระบบตัดรอบตรวจสอบ: ก่อนเที่ยงคืน
→ ไม่พบการปิดสิทธิ์ → ติด C305
```

บริการที่เสี่ยงสูง:
- **บริการฟอกไต** — ใช้เวลาหลายชั่วโมง มักข้ามเที่ยงคืน
- **บริการฉุกเฉิน/เวรดึก** — ผู้ป่วยสิ้นสุดบริการหลังเที่ยงคืน
- **บริการผ่าตัด** — เวลานานและแปรผัน

## วิธีแก้ไข

### วิธีที่ 6 — พิสูจน์ตัวตน + ส่งข้อมูลภายในวันเดียวกัน

- ปิดสิทธิ์และส่งข้อมูลภายในวันที่บริการ (ก่อนเที่ยงคืนของวันนั้น)
- ระบบจะ **ขยายเวลาตรวจสอบเป็น 24 ชั่วโมง** โดยอัตโนมัติ
- หน่วยบริการไม่จำเป็นต้องเร่งส่งข้อมูลหลังเที่ยงคืน

### วิธีที่ 7 — Minimal Data Set

- ส่งข้อมูล Minimal Data Set ก่อน (6 fields)
- **ข้อจำกัด:** ถ้าอยู่ในช่วงคร่อมและยังไม่เปิดทำการ อาจส่งข้อมูลไม่ทัน

## กรณีพิเศษ — บริการฟอกไต

- **ข้อบังคับ:** ต้องปิดสิทธิ์ **หลังจากฟอกไตเสร็จเท่านั้น** ปิดก่อนไม่ได้
- มีข้อจำกัดเรื่องระยะเวลาให้บริการฟอกไตด้วย

## ข้อเสนอแนะจากหน่วยบริการ (สู่ สปสช.)

หน่วยบริการเสนอให้ สปสช. ปรับปรุงระบบรองรับบริการช่วงรอยต่อของวัน เช่น:
- กำหนดเงื่อนไขบริการข้ามวันชัดเจน
- ขยายเวลาตรวจสอบ Minimal Data Set สำหรับบริการที่ยังไม่เปิดทำการ

สปสช. รับข้อเสนอไว้พิจารณา
