# C305 / C30 — กระบวนการอุทธรณ์สำหรับหน่วยบริการนวัตกรรม

## Definition

กระบวนการอุทธรณ์เพื่อขอรับค่าชดเชยสำหรับบริการที่ถูกปฏิเสธด้วยรหัส **C30** (ข้อมูลไม่ผ่านการตรวจสอบ) หรือ **C305** (เวลาปิดสิทธิ์ไม่สอดคล้อง) โดยเฉพาะกรณีที่เกิดจากปัญหาการเปลี่ยนผ่านระบบใหม่ในช่วงเดือนมกราคม 2569 สำหรับหน่วยบริการนวัตกรรม

## ช่องทางอุทธรณ์

**เว็บแอปพลิเคชัน:** `appel349.nhso.go.th`

**ระยะเวลาที่เปิด (รอบ ม.ค.–ก.พ. 2569):**
- เริ่ม: **1 กุมภาพันธ์ 2569**
- สิ้นสุด: **28 กุมภาพันธ์ 2569**

## เอกสาร 3 ส่วนที่ต้องแนบ

### ส่วนที่ 1 — หนังสือรับรองภาพรวม
- สรุปจำนวนเคสที่ถูกปฏิเสธทั้งหมด
- ระบุรหัสธุรกรรม (Plan ID) ของแต่ละเคส
- หน่วยบริการนำ Plan ID ไปจับคู่กับข้อมูลที่ส่งเบิก เพื่อระบุชื่อ-นามสกุล / เลขบัตรประชาชน 13 หลัก / วันที่รับบริการ

### ส่วนที่ 2 — รายละเอียดรายบุคคล
- แจ้งเหตุผลแยกรายบุคคล (แต่ละเคสอาจมีเหตุผลต่างกัน)
- ระบุสาเหตุที่ทำให้ถูกปฏิเสธในแต่ละราย

### ส่วนที่ 3 — หนังสือจากผู้มีอำนาจ
- ลายมือชื่อผู้มีอำนาจลงนามของหน่วยบริการ
- รับรองความถูกต้องของข้อมูลที่ส่งอุทธรณ์

## ขั้นตอนการพิจารณา

1. สปสช.เขตในภูมิภาคร่วมตรวจสอบความพร้อมของชุดข้อมูล
2. สปสช.ส่วนกลางจัดกลุ่มเหตุผล (Group by reason) ของทุกเคส
3. สปสช.พิจารณาเฉพาะเหตุผลที่มีความจำเป็นจริงๆ
4. **ไม่ใช่ทุก C30/C305 จะได้รับการชดเชย** — ขึ้นกับเหตุผลที่ยื่น
5. เคสที่อนุมัติ → ออก Statement ปกติ ระยะเวลาตรวจสอบ ~30 วัน

## กรณีที่ไม่ต้องอุทธรณ์ (สปสช.ดำเนินการให้อัตโนมัติ)

- บริการที่ถูกปฏิเสธเพราะ **FVS ล่มจากฝั่งสปสช.** (วันที่ 22 ม.ค. 2569 เป็นต้น) → สปสช.จะ waive และ reprocess
- บริการที่ **เลือกประเภทบริการผิด** ใน FVS → สปสช. reprocess ให้โดยอัตโนมัติ
- **C305 ที่เกิดจากระบบสปสช.เอง** (~700,000 records) → สปสช.แก้ให้อัตโนมัติ (ดู [[concepts/การปิดสิทธิ์]])

## ข้อมูลที่ใช้ตรวจสอบก่อนอุทธรณ์

- เข้า **Dashboard CD9** ดูรายการที่ถูกปฏิเสธรายหน่วยบริการ
- ดาวน์โหลด "C Deny Verify" จาก eClaim system → เมนูดาวน์โหลด → "รายละเอียดการผิดพลาดของข้อมูล"
