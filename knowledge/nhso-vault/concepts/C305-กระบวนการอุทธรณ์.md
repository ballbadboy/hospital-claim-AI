# C305 — กระบวนการอุทธรณ์และค้นหาข้อมูล

## Definition

**C305** คือรหัสปฏิเสธการจ่ายที่เกิดขึ้นเมื่อหน่วยบริการ **ปิดสิทธิ์ผู้ป่วยไม่ถูกต้อง** หรือ **ข้อมูลการปิดสิทธิ์ขาดหาย** ในระบบ สปสช. โดยในปีงบ 2569 สปสช. เปิดให้หน่วยบริการอุทธรณ์ C305 ผ่านระบบ Appeal 349 โดยเฉพาะ

## ระบบสำหรับอุทธรณ์

### URL หลัก

```
https://349.nhso.go.th
```

(Login ด้วย username/password เดิมของหน่วยบริการ)

## ขั้นตอนการอุทธรณ์ C305

### ขั้นตอนที่ 1: ค้นหาข้อมูลที่ติด C305

ข้อมูลแบ่งเป็น 2 กลุ่มตามระบบที่ส่ง:

#### กลุ่ม A — ข้อมูลใน eClaim (eKhlaim)
- เข้า `349.nhso.go.th` → ดาวน์โหลด IP ที่ระบบเตรียมไว้ให้
- สปสช. ตัดข้อมูล ณ **29 มกราคม 2569** และโหลดรายการล่าสุดให้หน่วยบริการ
- หากไม่พบ → สามารถค้นผ่าน eClaim โดยตรงและนำ Tran ID มาใช้

#### กลุ่ม B — ข้อมูลนอก eClaim (ระบบอื่น)
เข้า **DMS for Simulate** แล้วค้นรายงาน C305 แยกตามประเภท:

| ประเภทบริการ | รหัสรายงานใน DMS for Simulate |
|---|---|
| แพทย์แผนไทย (TT) | TT305 |
| Disability (ผู้พิการ) | รายงาน 305 ใน Disability |
| ฟื้นฟูสมรรถภาพ (DS) | DS305 |
| KTB (กองทุนทันตกรรม) | รายงาน C30 ใน KTB |

**หมายเหตุ TT305:** Tran ID จากระบบ OP Individual มีรูปแบบต่างจาก Tran ID ใน eClaim — ต้องใช้ Tran ID จากระบบ OP Individual ของ NHSO.go.th (เมนู → สำหรับหน่วยบริการ → บริการออนไลน์ → ชดเชย → OP Individual)

### ขั้นตอนที่ 2: จัดทำไฟล์ Excel อัปโหลด

**รูปแบบไฟล์ที่ต้องการ (2 คอลัมน์เท่านั้น):**

| คอลัมน์ | ชื่อ | รายละเอียด |
|---|---|---|
| A | Tran ID | รหัสธุรกรรมที่ติด C305 — **ต้องแปลงเป็น Text** (คลิกขวา → Format Cells → Text) |
| B | เหตุผล | เหตุผลที่ไม่สามารถปฏิบัติตามเงื่อนไขได้ — ระบุทุก record |

**ชื่อไฟล์แนะนำ:**
```
[รหัสหน่วย 5 หลัก][DD][MM][YYYY].xlsx
ตัวอย่าง: 1234506022569.xlsx
```
(ไม่บังคับ format ชื่อ แต่ต้องมี content ถูกต้อง)

**ข้อควรระวังพิเศษ:**
- คอลัมน์ A Tran ID **ห้ามเป็นตัวเลข (Number format)** — ต้องเป็น Text เท่านั้น มิฉะนั้นระบบจะ error
- ทุก record ต้องมีเหตุผลในคอลัมน์ B ครบถ้วน
- ถ้ามีหลายงวดจ่าย ให้ระบุแยกงวดที่หน้าเอกสาร

### ขั้นตอนที่ 3: อัปโหลดไฟล์

- Login เข้า `349.nhso.go.th`
- เลือกดาวน์โหลดข้อมูล C305 ของหน่วยตัวเอง (ระบบดึงตามรหัสหน่วยที่ login)
- กดอัปโหลดไฟล์ Excel ที่จัดทำไว้
- ระบบจะประมวลผลและแก้ไขข้อมูลโดยอัตโนมัติ

## สถานะการเปิด Appeal

- **ข้อมูลตั้งต้น:** สปสช. เตรียมข้อมูล C305 ถึงวันที่ 29 มกราคม 2569
- **ระยะเวลาแก้ไข:** เปิดรับ — ยังไม่มีการขยายเวลา (ณ 6 กุมภาพันธ์ 2569)
- **ข้อมูลปี 68:** สามารถอุทธรณ์ผ่านช่องทางเดียวกันได้ หากยังมีการค้าง

## หน่วยนวัตกรรม ที่ติด C305

หน่วยนวัตกรรม (คลินิกเอกชน) ที่ติด C305 → ค้นหาข้อมูลใน **DMS for Simulate** ได้เช่นเดียวกัน โดยนำ Tran ID จากระบบของแต่ละประเภทมาอัปโหลดใน Appeal 349 ได้เลย

## ปัญหาที่พบบ่อย

| ปัญหา | สาเหตุ | วิธีแก้ |
|---|---|---|
| ไม่พบข้อมูลใน Appeal 349 | ระบบเลือกรายการล่าสุด ไม่ใช่รายการแรก | ค้นหาเพิ่มเติมจาก eClaim หรือ DMS for Simulate |
| Upload แล้ว Error | Tran ID เป็น Number Format | แปลงคอลัมน์ A เป็น Text แล้ว upload ใหม่ |
| ข้อมูลจาก KTP/แผนไทย ไม่พบ | ต้องค้นจาก DMS for Simulate ไม่ใช่ eClaim | เข้า DMS for Simulate → รายงาน TT305 |
| อุทธรณ์แล้วยังติด E3/E5 | ข้อมูลยังไม่ครบหรือเหตุผลไม่ชัดเจน | ส่งข้อมูลรายละเอียดเพิ่มเติมทาง Hotline |

## ข้อมูลเพิ่มเติม (ครั้งที่ 84 — 16 ม.ค. 2569)

### เงื่อนไขสำคัญที่มาจากประกาศ พ.ร.บ. 2567

ประกาศคณะกรรมการหลักประกันสุขภาพแห่งชาติ เรื่องมาตรการในการดำเนินการเกี่ยวกับการเรียกเก็บค่าใช้จ่ายเพื่อบริการสาธารณสุขของหน่วยบริการ พ.ศ. 2567 (ประกาศพร้อมนโยบาย 30 บาทรักษาทุกที่) กำหนดว่า:
- หน่วยบริการ **ต้องจัดให้มีการแสดงตนยืนยันสิทธิ์** เมื่อสิ้นสุดการรับบริการ
- หากไม่ดำเนินการ → **หน่วยบริการไม่มีสิทธิ์ได้รับค่าบริการ** เว้นแต่มีเหตุจำเป็นที่หลีกเลี่ยงไม่ได้

### เงื่อนไขการอุทธรณ์ C305 (รอบ ก.พ. 2569)

| เงื่อนไข | รายละเอียด |
|---------|-----------|
| รหัสที่ติด | ต้องเป็น **C305 รหัสเดียวโดดๆ** ไม่มี C-Code อื่นร่วม |
| ช่วงเวลา | บริการในปีงบ 2568 ที่ยังค้างอยู่ |
| ระยะเวลาแก้ไข | **1–28 กุมภาพันธ์ 2569** |
| ช่องทาง | ระบบอิเล็กทรอนิกส์ (ระบบเดิมที่เคยใช้ปี 68) |

### เอกสารที่ต้องเตรียม

1. **หนังสือรับรองการให้บริการ** — ลงนามโดยผู้บริหารที่มีอำนาจของหน่วยบริการ (ใช้ประกอบการตรวจสอบของ สตง.)
2. **เหตุผลรายบุคคล** — ระบุเหตุผลที่ไม่สามารถปิดสิทธิ์ได้ แยกรายคนทุกรายที่ถูกปฏิเสธ
3. **Transaction ID** — สปสช. เตรียมข้อมูล C305 ใส่ระบบให้แล้ว (หน่วยไม่ต้องหา Tran ID เอง)

### สถานะการแก้ไขอัตโนมัติ (ปี 68)

| กลุ่ม | จำนวน | ผู้รับผิดชอบ |
|------|--------|------------|
| สปสช. แก้ให้อัตโนมัติ | ~700,000 รายการ | สปสช. (error จากระบบ) |
| หน่วยต้องแก้เอง | ~800,000 รายการ | หน่วยบริการ (ขาดการปิดสิทธิ์จริง) |
| รวมทั้งหมด | ~1,500,000 รายการ | — |

> ก่อนวันที่ 1 กุมภาพันธ์ สปสช. จะเชิญประชุมชี้แจงผ่าน Facebook Live อีกครั้ง เพื่อสาธิตวิธีใช้โปรแกรม
